Fora de Área: Um Jogo Educacional Móvel para Conscientização da População sobre os Riscos de Dirigir Manuseando Telefones Celulares
Descrição: Atualmente, é visível o crescimento da quantidade de veículos que transitam todos os dias nas ruas das cidades. Aliado a isto, infelizmente, há o aumento da quantidade de acidentes de trânsito. O uso de telefones celulares durante a condução de um veículo é considera uma das principais causas de acidentes de trânsito. Neste artigo, apresenta-se um jogo educacional móvel para a plataforma Android, denominado ?Fora de Área?, cujo enfoque está na conscientização dos indivíduos (não apenas motoristas), quanto aos riscos de dirigir manuseando telefones celulares.. OntoReq: Abordagem para Identificação de Interesses Transversais em Documentos de Requisitos com o Apoio de Ontologias de Domínio
Projeto certificado pelo(a) coordenador(a) Rosângela Aparecida Dellosso Penteado em 08/05/2013.
Descrição: Esta proposta tem por objetivo a criação de um processo semi-automatizado para identificação e classificação de interesses a partir de documento de requisitos com a utilização de ontologias de domínio. Interesses referem-se a requisitos de software que se encontram espalhados ou entrelaçados no produto final. Um software produzido com qualidade deveria ter um módulo específico que agrupa um conjunto de exigências relacionadas. Entretanto, há alguns tipos de requisitos para os quais essa clara alocação em módulos não é possível com a utilização somente das abstrações usuais de engenharia de software, como casos de uso, classes e objetos, entre outros. Tais requisitos são denominados ?requisitos transversais? ou mais comumente chamados ?early aspect? e dificultam o desenvolvimento e a manutenção do software por se apresentarem espalhados pelos demais módulos do software. O enfoque deste projeto está principalmente na identificação de interesses nas fases iniciais do ciclo de vida do software e na classificação desses interesses como base ou transversais, visando a facilitar o desenvolvimento e a evolução desse software. O processo manual também pode ser falho, assim, ferramentas para automatização de algumas das atividades do processo para identificação e classificação de interesses devem ser desenvolvidas. A avaliação do processo proposto bem como das ferramentas de apoio desenvolvidas será feita por meio de experimentos controlados.. Situação: Concluído; Natureza: Pesquisa. 2009 - 2011
Recuperação de Modelos de Classes Orientados a Aspectos a partir de Sistemas Orientados a Objetos usando Refatorações de Modelos
Descrição: Neste trabalho foi desenvolvido um conjunto de refatorações que pode ser aplicado sobre modelos de classes OO anotados com indícios de interesses transversais (ITs) para obtenção de modelos de classes OA. Modelos de classes anotados são diagramas de classes da UML cujos elementos (classes, interfaces, atributos e métodos) são anotados com estereótipos referentes aos ITs existentes no software. O conjunto de refatorações desenvolvido é subdivido em: i) refatorações genéricas; e ii) refatorações específicas. As refatorações genéricas são responsáveis por transformar um modelo OO anotado com indícios de ITs em um modelo OA parcial - modelo cujos ITs existentes nele não foram completamente modularizados. Essas refatorações são aplicáveis a qualquer tipo de IT existente no software, considerando o cenário que esses interesses se apresentam no modelo de classes. As refatorações específicas são responsáveis por transformar modelos de classes OA parciais em modelos de classes OA finais - modelos nos quais os ITs foram completamente modularizados em aspectos. Três refatorações genéricas e seis refatorações específicas para os interesses de persistência (subdividido em: gerenciamento de conexões, de transações e sincronização), de logging e para os padrões de projeto Singleton e Observer foram elaboradas. Um plug-in Eclipse, denominado MoBRe, foi desenvolvido para auxiliar o Engenheiro de Software na tarefa de aplicação das refatorações.. Situação: Concluído; Natureza: Pesquisa. 2008 - 2008
Critérios de Testabilidade para Avaliação do Modelo de Projeto de Software Orientado a Aspectos
Descrição: A atividade de teste de software é realizada visando a assegurar a maior qualidade possível do software. Apesar de ser impossível provar que um software é livre de defeitos, a aplicação de testes fornece evidências da conformidade com a funcionalidade especificada. Entretanto, sabe-se que as atividades relacionadas com os testes precisam ter o seu planejamento iniciado logo no princípio do ciclo de desenvolvimento, contribuindo para evitar defeitos e sua propagação nas demais fases do desenvolvimento. Desta forma, este trabalho propõe a avaliação do Modelo de Projeto de software orientado a aspecto, considerando as características de testabilidade. Para isso, foi definido um conjunto de critérios de testabilidade para ser usado na avaliação. Estes critérios foram utilizados em um software real para verificar a sua aplicabilidade..
