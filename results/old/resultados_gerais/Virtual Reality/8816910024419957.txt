2014 - Atual
Geração automática de dados de teste para cobrir caminhos dependentes de exceção usando execução simbólica
Projeto certificado pelo(a) coordenador(a) Marcelo Medeiros Eler em 28/01/2015.
Descrição: O teste de software é uma atividade chave para a garantia de qualidade de software. Técnicas e estratégias de teste auxiliam os testadores na criação e na avaliação de casos de teste. A execução simbólica tem sido utilizada como uma técnica efetiva para a geração automática de dados de teste para cobrir os requisitos de teste definidos pelo critério de fluxo de controle da técnica de teste estrutural. Na execução simbólica, cada caminho do programa sob teste é representado por uma sequência de restrições que deve ser satisfeita palas variáveis de entrada para que o caminho seja executado. Valores concretos são definidos para cada variável por meio de solucionadores de restrições (constraint solvers), tal que todas as restrições sejam satisfeitas. Os valores das variáveis são então usados como dados de teste para executar cada caminho. A maioria das abordagens explora apenas técnicas para gerar dados de teste para caminhos livres de exceção, que são caminhos nos quais os mecanismos de tratamento de exceção não são levados em consideração. A geração de dados de teste para caminhos dependentes de exceção, que são os caminhos que são executados apenas quando uma dada exceção é lançada, não tem sido tratada apropriadamente pelas abordagens atuais. Essa limitação existe porque as restrições para que uma exceção seja lançada não estão sempre explícitas no código, a não ser que sejam declaradas pelos desenvolvedores. Se não houver restrições definidas pelo usuário, somente uma análise do código sob a ótica de cada tipo de exceção pode revelar quais dados de teste podem provocar a exceção. Portanto, este projeto de pesquisa tem o objetivo de investigar a geração de dados de teste para caminhos dependentes de exceção e propor técnicas para definir restrições capazes de garantir que as exceções necessárias sejam lançadas e evitar que as exceções indesejadas sejam ativadas.. Situação: Em andamento; Natureza: Pesquisa. Integrantes: Vinícius Humberto Serapilha Durelli - Integrante / André Takeshi Endo - Integrante / Marcelo Medeiros Eler - Coordenador.
2012 - Atual
Empirical Software Engineering for Critical Embedded Systems
Projeto certificado pelo(a) coordenador(a) Elisa Yumi Nakagawa em 21/01/2016.
Descrição: Critical embedded systems (CES) have become increasingly important in today's society. However, designing these systems is still notoriously challenging. The main challenge is that these systems have to meet certain quality attributes, e.g., safety, dependability, and security. Failing to do so may incur significant physical damage or even loss of human lives. A promising solution to cope with this challenge is to design a sound architecture whose design decisions are meant to support certain quality attributes. 
Therefore, it makes sense to make sure that a given architecture is the right one. However, evaluating whether a given architecture achieves all proposed quality attributes is a complex undertaking. Moreover, there is no architecture evaluation method tailored to CES.  Although existing evaluation methods have served well in many domains, we believe that they should be shaped to the needs of CES. Thus, we set out to devise a lightweight ranking method to evaluate the most fundamental elements that underpin software architectures, namely, design decisions..
