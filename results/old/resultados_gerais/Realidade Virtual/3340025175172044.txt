2011 - 2014
CanCore-OBAA: uma proposta de especificação para metadados de Objetos de Aprendizagem Brasil-Canadá
Descrição: Este projeto de pesquisa visava propor um conjunto de metadados para objetos de aprendizagem a partir das especificações CanCore (desenvolvida no Canadá) e OBAA (desenvolvida no Brasil). Os objetos de aprendizagem são recursos didáticos digitais, organizados em repositórios, através de arquivos metadados responsáveis por descrever as propriedades técnicas, pedagógicas e de autoria. O CanCore e o OBAA foram desenvolvidos com base na especificação IEEE LOM, entretanto, cada um deles incluiu as necessidades e particularidades de cada país. O CanCore considera o sistema educacional canadense para especificar os objetos de aprendizagem. O OBAA possibilita a descrição de conteúdos multiplataforma e segmentados, além de requisitos de acessibilidade e comunicação com sistemas multiagentes, através de ontologias. Logo, através deste projeto, objetiva-se estabelecer uma nova especificação, buscando a convergência entre o CanCore e OBAA. Como resultado desta pesquisa, tornou-se mais fácil a busca de objetos de aprendizagem em repositórios de instituições brasileiras e canadenses, contribuindo assim, para o intercâmbio dos recursos didáticos.. Situação: Concluído; Natureza: Pesquisa. Integrantes: Mauricio Covolan Rosito - Integrante / Júlia M. 2009 - 2014
Reconfiguração Dinâmica em Projetos de Desenvolvimento de Software
Descrição: Desenvolver um produto de software requer um esforço singular e envolve lidar com atividades e artefatos para se obter os resultados desejados. Este projeto de pesquisa tem seu foco na integração das áreas de Engenharia de Software e Gerência de Projetos de Software. Sabe-se que o desenvolvimento de sistemas de software está sujeito a muitos tipos de incertezas que costumam ser mais relacionadas com questões gerenciais do que técnicas, tais como escopo, tempo, custo e restrições de recursos. Por outro lado, há uma variedade de processos de desenvolvimento de software disponíveis na literatura, os quais buscam sistematizar as ações necessárias para se produzir um produto e controlar o seu processo. Sabe-se, também, que, para se desenvolver um produto com qualidade, é necessário combinar adequadamente essas duas dimensões. Esta integração, contudo, não tem sido adequadamente coberta pelos processos de desenvolvimento de software atualmente disponíveis. Neste contexto de integração, pode-se trazer o conjunto de conhecimentos em gerência de projetos disponível no Guia do PMBOK (Project Management Body of Knowledge) como a perspectiva gerencial (escopo e ciclo de vida do projeto), enquanto modelos tais como o RUP (Rational Unified Process) e o OPEN, entre outros, podem ser usados para proporcionar a perspectiva técnica (escopo e ciclo de vida do produto).. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (1)  / Mestrado acadêmico: (1)  / Doutorado: (2) .
