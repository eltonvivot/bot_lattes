2021 - Atual
Resolução de Entidades para Big Data no setor de Saúde
Descrição: A Resolução de Entidades (RE), ou seja, a tarefa de identificar entidades semanticamente similares que se referem a um mesmo objeto do mundo real, em múltiplas fontes de dados, é uma tarefa importante e desafiadora para integração e análise de dados. Exemplos de entidades incluem pacientes, pesquisadores e remédios. As entidades podem estar representadas em formato estruturado (e.g. registros de bancos de dados) e semiestruturado (e.g. JSON). No setor de Saúde, a tarefa oferece suporte para a realização de estudos epidemiológicos, análise de dados longitudinais, avaliação de políticas sociais,entre outros. Uma das maiores dificuldades para a realização da RE é o tempo de execução elevado proveniente da natureza quadrática da tarefa. No contexto de Big Data, este desafio 
torna-se ainda maior uma vez que o volume de entidades contidas nas fontes de dados parte da ordem de milhões. Este projeto tem como foco a concepção de métodos de RE envolvendo fontes de dados do setor de Saúde. Para lidar com Big Data, os métodos devem contemplar as seguintes características: volume, variedade e velocidade. Para atender ao requisito volume de dados, será proposto um método para execução da tarefa de RE em paralelo em ambientes computacionais distribuídos com o uso de modelos de programação como Spark. No tocante ao requisito variedade de formatos de representação de dados, será
proposto um método de RE que ignora os atributos das entidades (esquema), ou seja, que considera apenas os dados em si. No que se refere ao requisito de velocidade de atualização de dados, será proposto um método de RE incremental, ou seja, um método que, a cada atualização da fonte de dados, executa a RE apenas na porção de dados atualizada. As soluções propostas serão validadas em estudos de caso reais no setor de Saúde, garantindo impacto no estado da prática, além do avanço no estado da arte, servindo como base para futuros investimentos em pesquisa na área.. OntoFrame: Um Framework para Gerenciamento de Ontologias em Processos de Integração Semântica de Esquemas
Descrição: O uso intensivo de dados disponíveis na Web, assim como a necessidade de encontrar informações mais precisas, promoveu um novo conceito, a Web Semântica, e fez surgir a necessidade de uma representação semanticamente mais rica dos dados. Neste contexto, a linguagem XML torna-se insuficiente como formato de representação de dados e os metadados passam a ser fundamentais neste processo de integração semântica. Surgem então novos formatos de representação de dados na Web, como RDF/RDFS, assim como o uso de ontologias, representadas em OWL, como forma de compartilhar conceitos de um determinado domínio. No sentido de enriquecer semanticamente seu formato de representação, os sistemas de integração de fontes de dados heterogêneas passaram a usar ontologias como forma de representar seus esquemas conceituais. A integração semântica destes esquemas conceituais gera a necessidade de mecanismos específicos para manipulação das ontologias, tais como comparação (matching), fusão (merging) e sumarização de ontologias. Neste sentido, considerando ambientes dinâmicos de integração semântica de dados, nossa proposta é a criação de um framework que possa ser utilizado em diversos tipos de ambientes de integração de dados. O objetivo deste projeto é a especificação e implementação deste framework para facilitar o uso dos diversos processos de manipulação de ontologias no desenvolvimento de sistemas que usem ontologias como modelo de representação de dados. O framework proposto será instanciado em um PDMS (Peer Data Management System), sistema de gerenciamento de dados em ambientes P2P (Peer-to-peer). Situação: Concluído; Natureza: Pesquisa. 2008 - 2009
SPEED - Uma Abordagem Semântica para Gerenciamento de Dados em Ambientes Peer-to-Peer (P2P
Descrição: O desenvolvimento de aplicações na Web fez crescer a necessidade por novas arquiteturas de sistemas distribuídos e de alto desempenho de processamento. Uma destas arquiteturas é a arquitetura Peer-to-Peer (P2P) que vem sendo bastante utilizada para trocas de informações, especialmente arquivos, na Web. Nosso interesse se concentra no problema de gerenciamento de dados, acessados a partir de fontes de dados distribuídas, e a integração dos mesmos para responder às consultas efetuadas nos diversos pontos que compõem um sistema P2P. Os sistemas de gerenciamento de dados P2P representam uma evolução dos tradicionais sistemas de integração de dados. Neste sentido, propomos a criação de um sistema utilizando uma arquitetura de super-peers baseada em uma abordagem semântica. Na arquitetura proposta, os pontos são agrupados de acordo com seu domínio semântico formando o que chamamos de clusters semânticos. Os clusters semânticos de um mesmo domínio formam uma comunidade semântica que agrega informações relevantes para o domínio como, por exemplo, uma ontologia de termos do domínio para facilitar o mapeamento entre os esquemas. Os principais pontos a serem considerados no desenvolvimento do sistema são: gerenciamento das conexões entre os pontos, geração e manutenção de mapeamentos semânticos, execução de consultas distribuídas, definição, e avaliação de critérios de qualidade de dados e informações. O projeto tem por objetivo a geração de um protótipo, implementando as soluções propostas em cada um dos pontos indicados. Situação: Concluído; Natureza: Pesquisa.
