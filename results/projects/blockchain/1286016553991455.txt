CINEMA: Computação INteligente, Escalável, Móvel e Altamente disponível na Internet
Projeto certificado pelo(a) coordenador(a) Elias Procópio Duarte Júnior em 03/08/2022.
Descrição: Descrição: O projeto CINEMA consiste de cinco frentes de trabalho inovadoras para a Internet. O projeto tem como principal alicerce a revolução que as tecnologias de virtualização já estão trazendo para a Internet: um movimento que está, acreditamos, apenas começando. No contexto de tecnologias de virtualização, o foco principal do projeto está na tecnologia NFV - Network Functions Virtualization (Virtualização de Funções de Rede). A primeira frente de trabalho do projeto tem como foco a alta disponibilidade de funções e serviços virtualizados de rede, que são composições que consistem da orquestração de múltiplas funções virtuais para a composição de serviços complexos de rede. A motivação para a alta disponibilidade é evidente: o movimento de substituição de soluções em hardware por software baseado em virtualização só vai realmente ter sucesso se as soluções apresentarem os níveis de robustez necessários. A segunda frente de trabalho, denominada NFV_inc, consiste em permitir que a própria infra-estrutura de rede ofereça serviços distribuídos inovadores implementados com tecnologia NFV. A terceira frente de trabalho tem como foco a investigação de técnicas de Inteligência Artificial Distribuída para a gerência de ecossistemas NFV. A quarta frente de trabalho é voltada para algoritmos distribuídos baseados na topologia virtual VCube, que permite a construção de soluções distribuídas eficientes e escaláveis. Por fim, a quinta frente tem foco em mobilidade com a arquitetura O-RAN (Open Radio Access Network), visando a implementação de componentes robustos aplicados no contexto de redes 5G.. Implementação e Avaliação do Comportamento entre Ferramentas para Detecção de Vulnerabilidades em Código Fonte para Aplicações Web
Descrição: Este trabalho propõe um firewall de aplicação que auxilia no
controle de segurança para aplicações Web através da implementação de
uma whitelist. Propõe-se também a utilização deste firewall no desenvolvimento e
manutenção das aplicações Web durante seu ciclo de vida e de
desenvolvimento. Além disso, a ferramenta proposta trabalha de forma
híbrida (whitelist e blacklist) fornecendo ainda maior segurança para as
aplicações Web.. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (2) . Integrantes: Rogério Correa Turchetti - Coordenador / Celio Trois - Integrante / Walter Priesnitz Filho - Integrante / Alfredo del Fabro Neto - Integrante / Bruno Gonçalves - Integrante.

2007 - 2008
Aperfeiçoando o Serviço de Tolerância a Falhas em Ambientes de Alto Desempenho
Descrição: As bibliotecas existentes para programação em ambientes de alto desempenho pouco disponibilizam técnicas de tolerância a falhas. Em situações de falhas, a aplicação está sujeita à perda da computação já efetuada, à problemas de desempenho e falta de confiabilidade.
Entretanto, adicionar confiabilidade através de um serviço de detecção de defeitos pode gerar falsas suspeitas, devido ao sistema assíncrono.  O objetivo deste projeto é aperfeiçoar  um detector de defeitos adaptativo, o qual possui quatro preditores para adaptação das falhas detectadas e integrar este detector à Biblioteca para Programação Paralela (BPP).. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (4) . Integrantes: Rogério Correa Turchetti - Coordenador / Ana Paula Canal - Integrante.
2006 - 2007
Desenvolvimento de Mecanismos Tolerantes à Falhas para a BPP - Biblioteca para Programação Paralela
Descrição: Em sistemas distribuídos onde os processos estão localizados em diversos nodos da rede, uma falha poderá gerar um colapso geral no sistema. Neste sentido, faz-se necessário a inclusão de mecanismos tolerantes a falhas, com a finalidade de permitir que aplicações possam disponibilizar seus recursos mesmo na presença de eventuais falhas. 
Pelo fato de que aglomerados de computadores executam, em geral, tarefas demoradas, e a BPP - Biblioteca para Programação Paralela em sua primeira versão, somente possui primitivas básicas de comunicação e sincronização, uma falha neste ambiente irá gerar a perda de todo o processamento já executado. Este problema pode ser contornado incluindo mecanismos tolerantes a falhas, como por exemplo, detectores de defeitos.. Situação: Concluído; Natureza: Pesquisa.
