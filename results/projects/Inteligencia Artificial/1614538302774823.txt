Este projeto tem por objetivo principal investigar e propor processos e tecnologia para melhoria da qualidade de software produzido pelo Laboratório Central (LACEN) da Eletronorte, de acordo com o item de sugestão de projeto 145 da Eletronorte no seu programa de P&D no ciclo 2003-2004.

Assim, a sinergia da integração das organizações envolvidas tem o potencial de disseminar soluções que atualmente são restritas, além de avançar para novos níveis de maturidade nos processos adotados.. PROSOFT - Cooperação e Gerenciamento do Processo de Desenvolvimento de Software de Alta Complexidade
Descrição: Contexto e Objetivos do Projeto
O projeto proposto visa acrescentar novas características importantes ao ambiente de desenvolvimento de software PROSOFT cujo desenvolvimento vem sendo realizado no PPGC-UFRGS.
	O projeto PROSOFT envolve a construção de um ambiente de desenvolvimento de software que tem como principal objetivo integrar ferramentas para desenvolvimento formal de software, criando-se assim um laboratório de experimentação. As ferramentas do PROSOFT visam auxiliar o desenvolvedor de software desde a especificação de requisitos até a implementação do sistema. O uso de métodos formais é enfatizado na construção das ferramentas do ambiente PROSOFT com paradigma próprio construído dentro do projeto. 
	Recentemente, este ambiente vem sendo utilizado para experimentação e validação de diferentes técnicas, ferramentas e metodologias para apoiar de forma integrada os aspectos de interação cooperativa de profissionais no desenvolvimento de produtos de alta complexidade. Deste modo, o PROSOFT Cooperativo [REI 98a] surgiu como um componente do projeto PROSOFT que visa gerenciar usuários, estações, objetos e ferramentas (incluindo seus estados e versões) a fim de proporcionar uma maior cooperação entre os desenvolvedores na construção formal de software.
Com a criação desse novo componente tornou-se necessária a expansão das atividades proporcionadas por ele, tal como um sistema de apoio à decisão em grupo e uma ferramenta para o desenvolvimento de interfaces para sistemas com trabalho cooperativo.

PROSOFT - Projeto de Ambiente de Desenvolvimento de Software
Descrição: Este documento contém as propostas de pesquisa e desenvolvimento sobre o ambiente de desenvolvimento de software PROSOFT desenvolvido no CPGCC - UFRGS. 
	O PROSOFT tem como objetivo principal apoiar o desenvolvimento formal de software, fornecendo integração de dados, controle e de apresentação entre suas ferramentas. Sua construção foi influenciada pela estratégia data-driven [NUN 92], conceito de modelos [BJØ 78], tipos abstratos de dados, orientação a objetos (OO), e  método algébrico [WAT 91], dentre outros conceitos.
	Os componentes do PROSOFT são os ATOs - Ambientes de Tratamento de Objetos. Todo ATO especifica algebricamente um tipo abstrato de dados e é composto, essencialmente, de uma classe e de um conjunto de operações que atuam sobre os objetos dessa classe.  Além disso, a ICS - Interface de Comunicação de Sistema, provê o mecanismo básico de comunicação dos ATOs PROSOFT. O desenvolvimento de um sistema, sob a ótica PROSOFT, consiste na definição de um ou mais ATOs.
	Todo ATO possui um nome, e na sua definição são especificados a sua classe (instanciação), representada graficamente e definida através da composição de tipos de dados (descritos na próxima seção),  a interface das operações (indicada, na figura, pelos nomes das operações - op1, op2, op3 - seguidos dos seus argumentos) e a especificação (semântica) das operações [NUN 94].
	A principal evolução ocorrida no ambiente PROSOFT do ponto de vista da implementação consiste da reengenharia do ambiente utilizando a linguagem JAVA [SCH 97]. Devido às características do paradigma utilizado pelo PROSOFT e dos requisitos de um ambiente de desenvolvimento de software distribuído, a tarefa de construir esta nova versão é bastante complexa e deverá ser enfatizada neste projeto.
