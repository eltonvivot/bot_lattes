Descrição: O projeto ?MAPS ? O protótipo de software será desenvolvido com base em um método para adaptação de processos de desenvolvimento de software (MAPS) desenvolvido nesta pesquisa. O método compreende: (1) extensão do metamodelo SPEM 2.0 com a inclusão de novos elementos e relacionamentos; (2) definição das regras de boa-formação para definição de processos de software consistentes; e (3) definição das operações e regras de boa-formação para adaptação de processos de desenvolvimento de software.
 Esta pesquisa, que teve sua origem em trabalhos desenvolvidos sobre a orientação deste pesquisador [PER05] e [COS09], encontra-se atualmente integrado a uma tese de doutorado também sobre a orientação deste pesquisador. A tese, que tem como principal objetivo o desenvolvimento do método citado acima, generaliza os resultados encontrados nos trabalhos anteriores utilizando o metamodelo SPEM 2.0, que é definido pela OMG como principal template para processos de desenvolvimento de software. Adicionalmente, é parte integrante da tese o desenvolvimento de um protótipo de software que suporte automaticamente o método proposto. Neste sentido, a inclusão de um novo aluno de graduação através do projeto ?MAPS ? Um Método para Adaptação de Processos de Desenvolvimento de Software? visa dar suporte às atividades de modelagem e implementação desenvolvidas na tese de doutorado. 
De modo conciso, os objetivos a serem alcançados neste projeto compreendem:
?	aprofundar o estudo teórico sobre a adaptação de processos;
?	especificar e implementar um protótipo de software baseado em um método para adaptação de processos de software (MAPS), possibilitando assim, a devida configuração do ciclo de vida dos projetos de sof. Reconfiguração Dinâmica em Projetos de Desenvolvimento de Software
Descrição: Desenvolver um produto de software requer um esforço singular e envolve lidar com atividades e artefatos para se obter os resultados desejados. Este projeto de pesquisa tem seu foco na integração das áreas de Engenharia de Software e Gerência de Projetos de Software.
Sabe-se que o desenvolvimento de sistemas de software está sujeito a muitos tipos de incertezas que costumam ser mais relacionadas com questões gerenciais do que técnicas, tais como escopo, tempo, custo e restrições de recursos. Por outro lado, há uma variedade de processos de desenvolvimento de software disponíveis na literatura, os quais buscam sistematizar as ações necessárias para se produzir um produto e controlar o seu processo. Sabe-se, também, que, para se desenvolver um produto com qualidade, é necessário combinar adequadamente essas duas dimensões. Esta integração, contudo, não tem sido adequadamente coberta pelos processos de desenvolvimento de software atualmente disponíveis.
Neste contexto de integração, pode-se trazer o conjunto de conhecimentos em gerência de projetos disponível no Guia do PMBOK (Project Management Body of Knowledge) como a perspectiva gerencial (escopo e ciclo de vida do projeto), enquanto modelos tais como o RUP (Rational Unified Process) e o OPEN, entre outros, podem ser usados para proporcionar a perspectiva técnica (escopo e ciclo de vida do produto).
Inserido como uma etapa complementar a dois trabalhos de pós-graduação (um de mestrado e outro de doutorado), este estudo visa contribuir com estas pesquisas por meio de uma análise detalhada dos modelos atualmente disponíveis, pela construção de um modelo integrado e pela produção de um protótipo para demonstrar os resultados alcançados. Uma Proposta de Tailoring para Projetos de Software
Descrição: A implantação de um processo padrão de desenvolvimento de software nas organizações tem sido fortemente demandando pela indústria, e vários processos têm sido sugeridos por organizações e companhias internacionais. Contudo, a criação de um processo padrão de desenvolvimento de software não garante melhoria na qualidade, pois na maioria das vezes a utilização do mesmo processo para todos os tipos de projeto não é possível, isto porque, diferentes projetos possuem diferentes necessidades. A maneira de fornecer flexibilidade aos projetos sem deixar de padronizar é através do tailoring do processo padrão de desenvolvimento de software. Tailoring consiste em adaptar um processo adicionando, excluindo ou modificando alguns de seus elementos e relacionamentos sendo o processo resultante mais apropriado para o alcance das metas do projeto. Entretanto, é importante considerar que permitir adaptações sem controle a partir do processo padrão, efetivamente implica que nenhum padrão existe. Por isto, as organizações devem estar preparadas para fornecer algumas regras a serem seguidas durante o processo de tailoring. Estas regras devem garantir que a conformidade do processo padrão de desenvolvimento de software seja mantida, o que exige um esforço considerável por parte das organizações. Neste sentido, uma pesquisa foi desenvolvida com objetivo de identificar como devem ser tratados os elementos de um processo padrão de desenvolvimento de software durante sua adaptação. A pesquisa foi conduzida no âmbito de uma dissertação de mestrado e teve como resultado a definição de um metamodelo e regras de boa-formação para tailoring de processos de desenvolvimento de software baseados no Rational Unified Process. Este projeto visa o desenvolvimento de uma ferramenta para uso automático do metamodelo e regras propostos. O principal objetivo da ferramenta é a geração de processos de projetos específicos para projetos de software através do tailoring do processo padrão.. Situação: Concluído; Natureza: Pesquisa. MASP - Projeto e Desenvolvimento de Sistemas Multiagentes
Descrição: Este projeto teve como objetivo especificar uma linguagem de modelagem para sistemas multiagentes (SMAs) e o conjunto de atividades a serem desenvolvidas para a construção de um sistema baseado neste paradigma. A linguagem de modelagem tem por finalidade permitir a elaboração dos artefatos de software (documentação e diagramas) envolvendo as visões estática, funcional e dinâmica do sistema, compondo os seus modelos de especificação. As atividades descrevem o processo de elaboração para um SMA utilizando-se da linguagem de modelagem, envolvendo todo o ciclo de desenvolvimento do sistema.
Para o alcance destes objetivos, diversas atividades foram desenvolvidas, envolvendo:
-	Orientação a dissertações de mestrado no tema do projeto;
-	Orientação de trabalhos de conclusão de curso de graduação relacionados ao tema do projeto;
-	Oferecimento de uma disciplina específica sobre o tema no Programa de Pós-Graduação em Ciência da Computação - PPGCC da PUCRS;
-	Elaboração de relatórios técnicos;
-	Elaboração de artigos científicos;
-	Desenvolvimento de aplicações na forma de protótipos;
-	Reuniões de trabalho entre os envolvidos no projeto.
O trabalho desenvolveu-se no período previsto para o projeto, permitindo a proposição da linguagem de modelagem e sua aplicação em dissertações de mestrado e trabalhos de conclusão de cursos de graduação. Através desta aplicação, foi possível realizar-se refinamentos ao longo do desenvolvimento do projeto.
Outro importante aspecto a ser ressaltado envolveu a criação de disciplinas para a graduação e pós-graduação associadas à temática do projeto. Desta forma, os resultados do projeto foram sendo disseminados a comunidade acadêmica ao longo do projeto, havendo perspectiva de continuidade destas disciplinas, em particular no pós-graduação..
