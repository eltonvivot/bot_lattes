2017 - 2018
Ambiente de desenvolvimento visual para a linguagem de programação C
Descrição: É possível observar a preocupação de grande parte dos países desenvolvidos, principalmente os da América do Norte e da Europa, no que diz respeito ao ensino de linguagens de programação. Estes países já perceberam que o domínio destas tecnologias impacta substancialmente no seu desenvolvimento econômico. Assim, os mesmos já atuam como provedor de tecnologia para os países que não a dominam. No Brasil, entretanto, o ensino de linguagens de programação tem sido reservado primordialmente às pessoas que já estão no ensino superior. Apesar dos problemas que o sistema educacional brasileiro enfrenta, o ensino de linguagens de programação apresenta outros desafios, que são enfrentados também por países desenvolvidos. Estes problemas estão relacionados à capacidade de abstração do estudante ao tentar descrever um problema do mundo real no ambiente virtual e às dificuldades impostas pela sintaxe e estruturas de uma determinada linguagem de programação. O presente projeto tem como proposta principal desenvolver um ambiente de desenvolvimento visual, baseado na biblioteca Blockly, para a linguagem de programação C, no qual os alunos iniciantes na linguagem poderão trabalhar com uma ferramenta visual que seja equivalente a estrutura textual de um código fonte na linguagem C. Desta forma, espera-se evitar o impacto inicial, característico da linguagem, nos períodos iniciais de aprendizado. A partir disso, espera-se que os usuários da ferramenta aumentem o seu interesse no aprendizado de linguagens de programação e a sua capacidade de abstração de problemas que possam ser solucionados computacionalmente. Propp: 00220.1300.1666. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (2) . Integrantes: Jorge Lima de Oliveira Filho - Integrante / Leard de Oliveira Fernandes - Coordenador.

Desenvolvimento de Ferramenta Computacional para aquisição e transmissão de dados veiculares
Descrição: Desde o ano de 2010 (Conama, 2004), os veículos produzidos no Brasil dispõem de um canal de comunicação (OBD - On-board Diagnostic) voltado para prover informações e ajustes destinados à manutenção e diagnóstico. Nos automóveis, a utilização de sistemas computacionais voltado para monitoramento é fundamental para analisar o comportamento do veículo e garantir o funcionamento dentro de padrões que são exigidos pela legislação em vigor. Esse projeto busca desenvolver um software voltado para aquisição de dados automotivos por meio da interface OBD e geração de uma base de dados para monitoramento veicular. O projeto é fruto de parceria de cooperação técnica-científica entre a UESC e MAXTRACK, a qual estabelece dentre objetivos, o desenvolvimento de projetos de P&D em tecnologia da informação e automação. A execução do projeto será subsidiada financeiramente pela MAXTRACK. Propp: 00220.1300.1720. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (2) . 2017 - Atual
Um sistema de telemetria de baixo custo para veículos híbridos baseado em hardware e software abertos.
Descrição: A evolução da tecnologia nos últimos anos possibilitou a redução dos componentes eletrônicos aumentando significativamente o poder de processamento dos computadores e a redução expressiva do custo destes equipamentos, contribuindo assim, para o surgimento dos veículos híbridos, que são veículos autônomos (drones, carros, etc) com a possibilidade de ser controlado remotamente com auxílio de uma câmera de vídeo embarcada que transmite imagens em primeira pessoa para um operador remoto. Uma característica dos veículos híbridos é a necessidade do uso de uma estação base de controle para receber todas as informações (dos sensores), o vídeo da câmera e enviar as informações de controle, este procedimento é conhecido como telemetria. Este trabalho visa o desenvolvimento de um sistema de telemetria modular, de baixo custo, com software de código fonte aberto, para controladores de veículos híbridos que possibilite a sobreposição de informações do controlador embarcado no vídeo em primeira pessoa e que possa ser adaptado a qualquer sistema robótico. Um protótipo de baixo custo será criado e a implementação da arquitetura do software será avaliada levando em consideração dois aspectos: tempo de resposta entre a estação base e o veículo e facilidade de uso para o operador. Propp: 00220.1300.1671. Situação: Em andamento; Natureza: Pesquisa. Alunos envolvidos: Graduação: (4)  / Especialização: (0)  / Mestrado acadêmico: (0)  / Mestrado profissional: (0)  / Doutorado: (0) . 2012 - 2016
Configuração de ambiente real para avaliação de utility recebida de grids p2p
Descrição: Neste trabalho considera-se um sistema de grid p2p que provê múltiplos serviços para os usuários. Um mecanismo de incentivo promove colaboração entre os peers. Já foi mostrado que o uso de mecanismos baseados em reciprocidade em tais sistemas evita os free riders e, ao mesmo tempo, promove o agrupamento de peers que tenham interações mutuamente lucrativas. Por outro lado, uma característica que somente agora está sendo estudada é a seleção de um portfólio de serviços. Normalmente os peers estão sujeitos a uma limitação de orçamento que os força a prover somente um subconjunto dos serviços que podem ser providos. Claramente, o subconjunto de serviços selecionados impacta no lucro que o grid retorna ao peer pois cada serviço vai ter um custo diferente e retornar uma utility específica. Além disso, a utility gerada por um serviço é fortemente influenciada pelo comportamento dos demais peers que pode mudar ao longo do tempo[7]. Os trabalhos mais recentes nesta área fazem uso de modelos matemáticos complexos e/ou experimentos simulados a fim de gerar e validar resultados no domínio de computação utilitária em grids. Este trabalho visa o desenvolvimento de um ambiente para que os artefatos de simulação e os modelos matemáticos possam ser verificados em experimentos em um verdadeiro grid computacional, permitindo confrontar seus resultados teóricos com dados colhidos em ambientes reais.
