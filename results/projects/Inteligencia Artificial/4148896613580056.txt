2012 - 2013
Framework de Monitoramento e Gerenciamento para Planejamento de Capacidade e Gestão Dinâmica de Recursos Computacionais de Aplicações ERP Distribuídas em Nuvem
Descrição: Com o advento da Cloud Computing (Computação em Nuvem), grande parte dos desafios relacionados à confiabilidade, escalabilidade e elasticidade de um sistema computacional podem ser tratados de forma dinâmica e ágil. Isto permite atender as demandas por recursos computacionais (e.g., CPU, memória, disco) de uma empresa em função do crescimento ou redução do seu negócio. No entanto, para os provedores de serviços na Nuvem, principalmente na camada de IAAS (Infrastructure as a Service), um grande desafio é saber mensurar a real necessidade de seus clientes em termos de recursos computacionais a curto e médio prazo. O planejamento de capacidade na Nuvem serve tanto para o provedor de infraestrutura planejar sua necessidade de expansão quanto para o cliente adquirir a configuração de hardware adequada para a sua necessidade e,  porque não, pré-contratar configurações adicionais para situações de pico visando garantir o SLA (Service Level Agreement) do seu serviço. Neste contexto, o objetivo deste projeto é desenvolver uma solução de monitoramento e gerenciamento dos recursos computacionais (Hardware e Software) que podem servir de base para o  planejamento de capacidade de uma aplicação ERP em Nuvem Privada.. 2010 - 2012
SIGMA - Sistema de Informação Geográfica para Monitoramento Ambiental
Descrição: Num país de tamanho continental como o Brasil, a tarefa de monitorar as agressões ao meio ambiente de forma efetiva e em larga escala é ainda um problema em aberto tanto na academia quanto nas soluções comerciais. De fato, este é um problema estratégico para os governos e ONGs no movimento de preservação ambiental que paira no mundo e, naturalmente, estão em busca de soluções que os auxiliem nessa tarefa.  Em função disto, o objetivo deste projeto é desenvolver um sistema georeferenciado altamente escalável para monitoramento ambiental dos biomas terrestres (ex. cerrado, amazônia, mata atlântica). Esse sistema será capaz de processar um grande volume de dados espaciais para analisar e correlacionar, em tempo-real, eventos complexos tal como o aumento do desmatamento ou incêndios próximos a leito de rios, áreas de preservação ambiental, dentre outros.. 2008 - Atual
Plataforma de Middleware Geoprocessamento Distribuído e Escalável
Descrição: O mundo está se tornando cada vez mais georreferenciado. Base de dados com referências geográficas sobre postos de gasolina, farmácias, pontos turísticos, lojas, hotéis, locais de alta-periculosidade, dentre outros, estão cada vez mais comum e baratas. Iniciativas como a do Senso IBGE 2010, que está georreferenciando toda a pesquisa feita em campo, permitirá conhecermos a distribuição geográfica não somente dos artefatos físicos, mas também das questões sócias-econômicas-culturais da população brasileira como quantidade de pessoas e renda das famílias, grau de escolaridade, dentre outros.  Essas informações são estritamente importantes para o governo tomar decisões e para o mercado prover serviços LBS para os usuários. As possibilidades são infinitas. No entanto, para explorá-las, faz-se necessário um sistema que permita cruzar terabytes de dados georreferenciados com a localização de milhares de usuários/objetos, simultaneamente. Para isto, este projeto visa desenvolver uma plataforma de middleware de geoprocessamento escalável que implementa uma estrutura de indexação de dados espaciais de forma distribuída. Esta plataforma permitirá que seja feito, simultaneamente, centenas de cruzamento entre a localização de um objeto/pessoa com o mapeamento georreferenciado de lojas, rodovias, pontos turísticos, áreas de alta periculosidade visando descobrir eventos do interesse do usuário tais como promoções de produtos ofertados por lojas co-localizadas, desmatamentos próximos a rios, queimadas em áreas de preservação, etc. Esta plataforma servirá de base para desenvolvermos diversos tipos de aplicações LBS de larga escala.. 2003 - 2006
MoCA - Mobile Collaboration Architecture
Descrição: MoCA é uma arquitetura de middleware que disponibiliza um conjunto integrado de serviços e APIs para o desenvolvimento de aplicações sensíveis ao contexto para redes locais sem fio IEEE 802.11 e dispositivos portáteis, tais como PDAs e notebooks. Os serviços incluem a coleta periódica, o armazenamento e a disponibilização de dados sobre o contexto de execução de dispositivos móveis, tais como o nível de energia, a qualidade do enlace sem fio, a intensidade de sinal de RF dos pontos de acesso próximos, entre outros.
Nesse trabalho, eu atuei ativamente no projeto, definição e integração dos serviços da arquitetura MoCA;
- Projetei e desenvolvi em Java uma API para comunicação assíncrona baseada em eventos que provê mecanismos de comunicação flexíveis para redes móveis englobando, por exemplo, notificação baseada em expressão via UDP e TCP;
- Projetei e implementei em Java o serviço de provisão de contexto para processar e compartilhar o contexto dos dispositivos móveis (e.g., uso da bateria, rede) com aplicações a serem adaptadas de acordo com o estado corrente desses dispositivos;
- Desenvolvi uma aplicação em ANSI C para WindowsXP e Linux para monitorar as informações de contexto do dispositivo móvel, tais como uso da CPU, bateria disponível, lista de Access Points no raio de alcance do dispositivo, dentre outros;
- Além dos serviços supracitados implementei em Java uma API de protocolos de comunicação UDP e TCP, o serviço de configuração e o serviço de privacidade;
- Escrevi aproximadamente 50,000 linhas de código na implementação dos serviços da arquitetura. Situação: Concluído; Natureza: Pesquisa.
