Descrição: O presente projeto visa a criação de um sistema de simulação de cirurgia. Nesse sistema o usuário interage com controles em forma de pinça. Para parte de interação tátil das pinças será usada um hardware já desenvolvido pelos colaboradores do projeto. O intuito final dessa pesquisa é validar esse simulador como uma ferramenta de ensino de técnicas de cirurgia, a fim de aprimorar o aprendizado dos alunos dessa especialidade de estudo.. NEDGlove: Um sistema computacional para uso em sessões de reabilitação motora virtual e interativa
Descrição: O laboratório Natalnet (www.natalnet.br) possui uma longa experiência com projetos de realidade virtual [Rocha et. Al. 2011]. Atualmente Natalnet desenvolve um projeto para construção de uma luva de dados [Burdea et. Al. 2003] para apoio a colheita de produtos hortícolas. Os resultados da construção dessa luva de dados já renderam uma importante publicação [Aroca et. Al. 2013].
Para realizar testes de sensores da luva foi desenvolvido um ambiente virtual que simula uma mão humana. Esse ambiente de visualização foi desenvolvido com o Unity 3D [Goldstone 2010]. O Unity trate-se na verdade de um motor de jogos e por isso permite simular interações físicas com objetos de uma forma simples e prática. 
O conjunto de informações capturadas pela luva possui uma aplicabilidade clara para aplicações ligadas a reabilitação de pacientes com sequelas nas mãos. Ao capturar informações sobre o ângulo de fechamento dos dedos e os eixos de rotação do punho, essa luva pode produzir dados valiosos para estudos de fisioterapia e serem aplicados a sessões de reabilitação virtual
Apesar das possibilidades que podem ser alcançadas com o uso de tal dispositivo em sessões de fisioterapia, o uso dessa técnica não é por si só capaz de reverter o fato de as sessões típicas de fisioterapias serem implementadas por ações repetitivas e, de certo modo, enfadonhas para o paciente. Porém, o uso da interação com elementos virtuais em um jogo de computador pode minimizar esse aspecto e tornar realização da sessão uma tarefa prazerosa. Isso é possível graças à metodologia do ?gamefication? [Deterding et. Al. 2011] na qual elementos de design de jogos são aplicadas a outros tipos de softwares.
Baseado no fato de que o ambiente de visualização dos dados da luva já é desenvolvida em um motor de jogo e baseado no conceito de gamefication, pretendemos desenvolver nesse projeto interfaces de interação humano computador que se apresentem forma de um jogo de computador, a ser usado em sessões de fisioterapia. Os jogos serão desenvolvidos para atenderem o proposito de uma sessão típica de fisioterapia, mas as interações estarão disfarçadas de interação de jogo de computador. Como o dispositivo de entrada de dados será utilizado uma luva de dados a ser desenvolvida para o projeto.
O presente projeto visa criação de um sistema de hardware e software para uso em sessões de reabilitação virtual. A parte de hardware é composto por uma luva que possui um série de sensores acoplados a mesma, o que permite que esta luva capture informações sobre a mão e os dedos, tais como o ângulo de fechamento dos dedos e os eixos de rotação do punho. A camada de software do sistema será formada por um ambiente de visualização dos dados gerados pela luva, contando com um simulador de uma mão virtual (que exibirá no computador os movimentos descritos pela mão que está vestindo a luva) e um visualizador numérico e gráfico dos dados coletados pela luva (ângulo de fechamento dos dedos e os eixos de rotação do punho). Pretendemos ainda desenvolver nesse projeto um jogo de computador, a ser usado em sessões de fisioterapia, que será controlado pela paciente quando ele estiver usando a luva. Os jogos serão desenvolvidos para atenderem o proposito de uma sessão típica de fisioterapia, mas as interações estarão disfarçadas de interação de jogo de computador, a fim de aumentar o engajamento do paciente na realização da sessão.. Muse - Um sistema de criação automatizada de museus virtuais 2D e 3D
Descrição: O laboratório Natalnet [2] vem trabalhando há uma década com pesquisas na área de ambientes virtuais. Entre as muitas vertentes estudadas destacam-se os trabalhos sobre museus virtuais [3,4]. As pesquisas sobre museus virtuais foram desenvolvidos a partir da colaboração com a UFPB e UERN através do Grupo de Trabalho de Museus Virtuais (GTMV) [1]. 
Grupo de Trabalho de Museus Virtuais trabalhou na criação de uma ferramenta para dar suporte a criação de museus virtuais 3D. Apesar de ser uma interface mais rebuscada do ponto de vista gráfico, os ambientes 3D ainda são minoria no uso para museus virtuais se comparados com os ambientes 2D (páginas web). As páginas web que se intitulam como museus virtuais apresentam em sua grande maioria figuras (fotografias ) das obras expostas em galerias de artes reais. Apesar dos esforços do GTMV para estabelecer um novo paradigma para o desenvolvimento de museus virtuais, facilitando a criação dos museus 3D, essa facilidade se mostrou aquém da facilidade de criação de uma página web. 
Ao trabalhar na concepção do Museu (virtual) de Arte Abraham Palatnik percebemos que mesmo a criação de uma página web (2D) para expor as obras de um acervo é um trabalho árduo e lento. Esse trabalho envolve programação web, consultas a banco de dados e processamento de imagens. Deixando o processamento de imagens aos cuidados de uma equipe de artes, tivemos um trabalho de três meses entre a concepção, design, programação, validação e implantação da página, contando com uma equipe de três pessoas. O mesmo tempo foi gasto para concepção do ambiente 3D contando com o dobro da equipe web. 
Percebemos, entretanto que o processos de criação da páginas web do Museu (virtual) de Arte Abraham Palatnik poderia ser otimizados baseados no reuso de porções de código que se repetiam em diversas páginas criadas. Essas porções de códigos chamadas aqui de ?artefatos base? poderiam ser usadas para criar de forma automática a página web de outros museus. As obras previamente armazenadas em um banco seriam acessadas automaticamente pela página. Partindo do mesmo principio poderia ser criada uma ?galeria 3D base? que iria carregar automaticamente para dentro do seu espaço virtual as mesmas obras armazenadas no banco. Portanto propomos nesse projeto a criação de um sistema para automatizar a criação de museus virtuais 2D (páginas web) e 3D. Uma vez criados esses museus estão correlacionados, pois serão alimentados com informações do mesmo banco de dados. Para manter a consistência das informações será usado um sistema de cadastro de usuários, onde cada usuário estará associado a um acervo especifico.. Situação: Concluído; Natureza: Pesquisa. ELA Home Care: Monitoramento Remoto de Pacientes com Esclerose Lateral Amiotrófica
Descrição: A Esclerose Lateral Amiotrófica (ELA) é uma doença neurodegenerativa, caracterizada por fraqueza muscular progressiva que leva o paciente ao óbito, usualmente devido a complicações respiratórias. A degeneração dos neurônios motores inferiores e superiores, atingindo a medula espinhal, o tronco encefálico e o córtex motor resultam na definição da ELA como doença do neurônio motor. No momento que os músculos ventilatórios são comprometidos os indivíduos apresentam restrições pulmonares, caracterizadas por redução da Capacidade Vital (CV) e do Volume Corrente (VC) com, conseqüente, insuficiência respiratória crônica. A Ventilação Não-Invasiva (VNI) consta na administração de ventilação mecânica aos pulmões sem que haja a necessidade de vias aéreas artificiais, podendo ser oferecida ao paciente por meio de ventiladores mecânicos ou através de aparelhos denominados Pressão Positiva Bifásica nas Vias Aéreas (Bilevel). Sua aplicação possui diversos objetivos, dentre eles: a melhora na qualidade do sono, na qualidade de vida, na redução do desconforto respiratório e no trabalho ventilatório, nas trocas gasosas e, por fim, o prolongamento da sobrevida. Assim, a utilização da VNI nos pacientes com ELA tem sido empregada nos últimos anos com o objetivo de corrigir a insuficiência respiratória e, por conseguinte, melhorar a qualidade de vida e prolongar a sobrevivência destes indivíduos. Todavia os dispositivos que realização a ventilação mecânica ao longo processo podem perder calibração, neste sentido prejudicando a melhoria da qualidade de vida dos pacientes acometidos com a ELA.
Quando estes pacientes estão internados em ambiente hospitalar, a perda de calibração pode ser facilmente corrigida por técnico da área de saúde especialista responsável pelo paciente. No entanto, quando se trata de um paciente em casa, o contexto é outro, pois nem sempre é possível ter um técnico especialista que possa garantir que o equipamento estará sempre em sincronia com as necessidades do paciente com ELA, essa tarefa é realizada nestes casos por um familiar ou um cuidador. 
Neste contexto, o presente projeto de pesquisa visa desenvolver um dispositivo baseado na computação móvel e em sistemas embarcados que possibilite monitorar de forma on-line os ventiladores mecânicos, de forma a garantir que quando houver perda de calibração um técnico especialista em uma unidade hospitalar será notificado via celular que o paciente pode está passando por dificuldades. Com isso, contribuindo significativamente para melhoria da qualidade de vida destes pacientes.. Situação: Concluído; Natureza: Pesquisa. Sistema portátil para auxilio de reconhecimento, contagem e manipulação de objetos diversos
Descrição: Este projeto apresenta a proposta de desenvolvimento de um dispositivo portátil com a capacidade de identificar objetos em tempo real usando visão computacional. O dispositivo proposto consiste em um equipamento miniaturizado que pode ser "vestido" no braço ou na mão do usuário, na forma de uma luva ou um bracelete. Tal projeto possui diversos desafios científicos e tecnológicos já que será necessário desenvolver técnicas de visão computacional em sistemas embarcados com baixo consumo de energia, para economia de bateria, além de sistemas de estabilização de imagem, pois a câmera não será fixa, mas acompanhará o movimento da mão ou braço do usuário. Enquanto o usuário move-se o sistema deve processar as imagens da câmera usando técnicas como shape-from motion [Trucco 1998] para estimar características da imagem e suas informações 3D, como distância, área, volume e forma de determinados objetos. Além de uma micro-câmera posicionada no aparelho, o sistema ainda possuirá comunicação wi-fi e possibilidade de interagir com sistemas de computação em nuvem [Reese 2009], bem como outros equipamentos do mesmo tipo próximos a ele, formando uma rede mesh [Feilong et al 2007]. A principal aplicação prevista é o apoio a colheita de produtos hortícolas em campo (frutas, vegetais e legumes em geral). Especialistas em agronomia apontam que a perda destes produtos desde a colheita até o consumidor final chega a 50% em países como o Brasil [Kader 2002], sendo que um dos principais problemas nesta cadeia é a classificação dos alimentos. De fato, as várias etapas de classificação que ocorrem em campo, no armazém, na distribuição, nos atacadistas e no varejo vão danificando aos poucos o fruto e contribuindo para que ele estrague. O uso do dispositivo proposto para a classificação poderá agilizar este processo, ajudando na redução do desperdício, já que a classificação no momento da colheita reduzirá a necessidade de etapas posteriores de classificação.. 2011 - 2012
Inclusão Digital baseada no desenvolvimento de ambientes virtuias interperceptivos
Descrição: Antes do surgimento da internet, os computadores se interligavam através de BBS?s (Bouletin Board System) que permitiam que seus usuários pudessem trocar informações remotamente. No final da década de 70, Roy Trubshaw e Richard Bartle, dois pesquisadores da Universidade de Essex, utilizaram uma BBS para executar um jogo onde vários usuários conectados interagiam através de uma interface textual. O jogo combinava elementos de RPG (Role-Playing Games) e interação social através de salas de bate-papo (chat). Ele foi batizado de Multi-user Dungeon [Bartle 1990] em referencia ao jogo de RPG Dungeons and Dragons [Gygax 1987], porem ficou mais conhecido pelo acrônimo MUD. O MUD foi o marco inicial para criação de ambientes virtuais multi-usuários on-line.
Nas décadas seguintes surgiram ambientes com interface 2D, entre eles merece destaque o Habitat [Morningstar & Farmer 1991] da LucasFilm Games. Depois veio a geração das interfaces 3D, onde merece destaque o Active Worlds [ActiveWorlds.com n.d.], que foi originalmente concebido para ser um navegador 3D da internet.
Atualmente existem ambientes virtuais com estes três tipos de interface (textual, 2D e 3D). O surgimento de ambientes com interfaces gráficas mais elaboradas e com mais dimensões não extinguiu os ambientes com interfaces mais simples. Ainda hoje podem ser encontrados MUD?s funcionando na internet [MUD n.d.]. Do mesmo modo, continuam surgindo ambientes com interface 2D, mas agora com suporte a uma quantidade massiva de jogadores. Esta situação pode ser explicada pelo fato que interfaces mais complexas exigem mais requisitos de hardware e nem todos os usuários possuem tais requisitos. Por isso, esses usuários buscam ambientes com interfaces mais simples.
Tendo em vista a existência de diversos tipos de usuários e de varias interfaces para interação com ambientes virtuais, propomos neste trabalho a criação de uma arquitetura que integre ambientes criados com interfaces distintas. Utilizando um sistema projetado com base nesta arquitetura, usuários conectados através das diferentes versões de um mesmo ambiente podem perceber e interagir uns com os outros de modo transparente.
A este processo de integração de ambientes de dimensões distintas demos o nome de interdimensionalidade. A técnica utilizada para permitir que os usuários percebam uns aos outros de modo transparente demos o nome de interpercepção.
Os sistemas interperceptivos permitem o acesso a um público maior de usuários que em muitos casos não teriam acesso a uma variedade de sistemas se não dispusessem de computadores com a configuração mínima para executá-los. Pelo poder de abranger uma quantidade maior de público, tornar um sistema interperceptivo abre novas perspectivas de negócios que antes só podiam existir para uma quantidade mais restrita de consumidores.
Neste projeto, nós propomos a criação de aplicativos que usam a Interpercepção [Dantas et al. 2011] para criar ambientes virtuais mais acessíveis. A interpercepção é uma abordagem que inclui um conjunto de regras e uma arquitetura de software que une usuários de interfaces diferentes no mesmo ambiente virtual. O sistema detecta os recursos do usuário e realiza transformações nos dados para sua visualização nas interfaces 3D, 2D e textual (1D). Isto permite a qualquer usuário se conectar, acessar todas as informações, e trocar informações com os outros usuários de um modo flexível, sem necessitar de alterações no hardware e software.. GIGA-VR
Descrição: Descrição: O objetivo principal deste projeto é desenvolver uma infraestrutura envolvendo componentes de hardware e software que possibilite a utilização remota de recursos de Realidade Virtual disponibilizados através de serviços que vão desde visualização até manipulação remota dos mesmos. Nosso intuito é possibilitar que instituições geograficamente dispersas possam usufruir recursos de última geração através da rede GIGA viabilizando o desenvolvimento de diferentes aplicações. Este projeto visa desenvolver e utilizar tecnologias que possibilitam a mistura entre o real e o virtual, segundo a abordagem utilizada em ambientes de Realidade Mista e Realidade Aumentada, cuja repercussão científica têm sido muito intensa nos últimos anos. Propomos o desenvolvimento de uma infra-estrutura que suporte vários tipos de conteúdos, com a implementação de um mecanismo de transmissão bidirecional da informação envolvendo texto, som, imagem, estruturas de dados tridimensionais, movimento usando rede de alta velocidade, ou apenas de seus parâmetros, no caso de usar-se rede normal. A transmissão de informação com tal volume e diferenciação dos conteúdos exige a definição de protocolos, métodos e técnicas apropriados. Mais do a utilização remota de recursos de RV (o que já uma tarefa complexa) pretendemos estabelecer uma infraestrutura que possibilite a criação de um espaço totalmente sintético envolvendo plataformas móveis de exibição/aquisição dos conteúdos e uso/manipulação de técnicas/equipamentos de realidade virtual. O desenvolvimento destes novos conceitos e deste tipo de plataforma constitui o problema principal focalizado pelo projeto que está sendo proposto. Para tanto, estamos apresentando uma infraestrutura que aglutine elementos de hardware, software e uma rede de transmissão (a rede GIGA) no intuito de suportar os requisitos necessários...
