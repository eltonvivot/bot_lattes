Interface para Sistema Integrado de Análise de Dados em Saúde
Descrição: A Ciência de Dados tem promovido grandes revoluções na área de negócios, principalmente auxiliando na recomendação de clientes, melhoria de logística, aumento de vendas e redução de custos. Recentemente tem-se identificado vários benefícios em analisar dados na área de saúde. Aplicações como coletar e organizar dados fornecidos por dispositivos de monitoramento pessoal, análise de prontuários médicos eletrônicos, auxílio para diagnóstico e laudos de exames e auxilio para definição de políticas de saúde coletiva são aplicações cada vez mais relevantes para a área de saúde. Além da complexidade dos dados em si, normalmente é necessário lidar com enormes quantidades de informações que suplantam as capacidades dos computadores atuais. Esse projeto tem como objetivo estudar e projetar uma interface gráfica de ferramenta para analisar grandes quantidades de dados de saúde e auxiliar as atividades dos profissionais de saúde. Para isso será construído um protótipo de interface gráfica, integrando várias ferramentas Open Source, que realize o tratamento das informações, analise suas características e construa um modelo para predição de resultado utilizando aprendizado de máquina. Esse projeto conta com a colaboração de pesquisadores dos cursos de pós-graduação na área de saúde da Universidade Estadual do Ceará (UECE) para auxiliar na definição dos problemas e na validação dos resultados. Como resultado final, espera-se disponibilizar uma interface de sistema para análise de grande volume de dados adaptado para tratar dados de saúde como ferramenta para a melhoria do ecossistema de saúde do país, tanto para o sistema público como para o sistema privado.. ContentSDN: Uma arquitetura de distribuição de conteúdo em Redes SDN
Descrição: Nos últimos anos, tem havido uma tendência no sentido de mudar o paradigma da Internet baseada na  localização para baseados em conteúdo. Com base no princípio de que as redes devem permitir que os usuários busquem os dados, em vez de ter uma referência a um local físico onde esses dados estão armazenados, sugere a adoção de novos paradigmas. Information-Centric Networking (ICN) é um paradigma que procura oferecer uma arquitetura de rede que melhora a distribuição de conteúdo. Estas melhorias, tais como o armazenamento de conteúdo em cache local, reduz atrasos e aumentar a velocidade de entrega de conteúdo multimídia como áudio e vídeo.. As Redes Definidas por Software, ou Software DSefined Networking (SDN), define uma nova arquitetura da Internet, onde dispositivos de rede simples e o plano de controle centralizado, fornece uma arquitetura de rede barata e confiável. A Arquitetura SDN oferece recursos poderosos para alterar e manipular pacotes de rede que permite implementar funções avançadas de rede. Este projeto propõe o ContentSDN,, uma arquitetura que integra ICN e SDN, fornecendo cache em rede transparente para distribuição de conteúdo como uma Função de Rede Virtualizada, Virtualized Network Function (VNF). ContentSDN permite orquestração de rede e alta escalabilidade em tempo de execução. Os cache de conteúdo são gerenciados por políticas  ​​que podem ser ajustadas de acordo com as necessidades do negócio. A arquitetura proposta será  avaliada em um ambiente emulado, onde muitos usuários requisitam conteúdo através do protocolo HTTP de uma forma transparente melhorando a eficiência de uso dos recursos de rede.. Detecção de Fluxos em Conflitos em Redes OpenFlow usando Lógica de Primeira Ordem
Descrição: A arquitetura OpenFlow é uma proposta da iniciativa Clean Slate para definir uma nova arquitetura da Internet, onde os dispositivos de rede são simples, e o plano de controle e gerenciamento é realizada por um controlador centralizado. A simplicidade e centralização tornam a arquitetura confiável e barata. No entanto, a arquitetura Openflow não fornece mecanismos para detectar fluxos conflitantes recebidos, permitindo que os caminhos inacessíveis podem ser configurados nos elementos de rede, e a rede pode não se comportar conforme o esperado. Este projeto propõe uma abordagem para detecção de conflitos utilizando a lógica de primeira ordem para definir possíveis antagonismos e usando um motor de inferência permite detectar fluxos conflitantes antes que o controlador OpenFlow configure os elementos da rede. A proposta foi implementada no controlador Floodligth e em um protótipo foi possível encontrar e eliminar conflitos em tempo real, evitando erros e melhorando o desempenho da rede.. PonderFlow: Uma Linguagem para especificação de Políticas em Redes SDN/OpenFlow
Descrição: A arquitetura SDN/OpenFlow é uma proposta da iniciativa Clean Slate para definir uma nova arquitetura da Internet, onde dispositivos de rede são simples, e o plano de gestão e controle são realizados em um controlador centralizado, chamado controlador OpenFlow. Cada controlador OpenFlow fornece uma Application Programming Interface (API) que permite que ao administrador de rede definir o tratamento desejado para cada fluxo que cruze a rede. No entanto, cada controlador OpenFlow tem sua própria API padrão, exigindo que os usuários definam o comportamento de cada fluxo em uma linguagem de programação ou scripting especifica. Essa arquitetura também torna difícil a migração dessas políticas de um controlador para uma outro, devido aos diferentes APIs. Este projeto propõe definir oPonderFlow, uma extensão da linguagem Ponder para especificação de política de rede em OpenFlow. O PonderFlow estende a linguagem de especificação Ponder originais permitindo definir uma regra de fluxo de OpenFlow abstratamente, independente do controlador OpenFlow usado. Alguns exemplos de política OpenFlow serão avaliadas mostrando sua sintaxe e a validação da gramática.. Situação: Concluído; Natureza: Pesquisa. Internet das Coisas (IoT) e Máquina a Máquina (M2M)
Descrição: Máquina a Máquina, ou do inglês, Machine-to-Machine (M2M), refere-se as diversas tecnologias que permitem tanto sistemas com fio quanto sem fio a se comunicarem com outros dispositivos de forma automática para atingir um objetivo comum. O M2M usa um dispositivo (um sensor medidor) para capturar um evento (como temperatura, pressão, umidade, luminosidade, etc), que é enviado através de uma rede (sem fio, com fio ou híbrida) para uma aplicação centralizada ou distribuída (programa), que transforma o evento capturado em informação útil (por exemplo, hora que pode desligar o ar condicionado e iluninação de uma sala). Isto é possível com o uso da telemetria, técnica para transmitir informaçãoe monitoradas. Entretanto, comunicações M2M extende-se além da conexão ponto-a-ponto entre sensor e servidores para sitema integrados com dispositivos pessoais para monitoração e comando (por exemplo, telefones, tablets, etc). A expansão das redes sem fio e da Internet em todo o mundo tornou possível o estabelecimento da comunicação M2M possibilitando a maior funcionalidade da tecnologia. A Internet nas Coisas (Internet of Things) refere-se a criar uma rede de objetos (coisas) se comunicando através de uma estrutura semelhante à Internet. O conceito da Internet das coisas requer uma forma de identificação dos objetos, e por isso, as etiquetas RFID é considerada muitas vezes como um pré-requisito para a Internet das Coisas. Se todos os objetos da vida cotidiana foram equipadas com etiquetas de rádio RFID, um computador poderia identificar e controlar todos os objetos. No entanto, a identificação única de objetos não é o bastante, podemos aplicar controles individuais aos objetos, por exemplo, desligar uma televisão ou acender as luzes do quarto.. Estudo de teste de Turing para segurança na Internet
Descrição: Provas Interativas de Humanidade, ou Human Interactive Proofs (HIP) estão cada vez mais presentes nos sistemas de computação atuais. A idéia por detrás delas é possibilitar que um determinado sistema de computação consiga distinguir se ele está sendo utilizado por um ser humano ou por um procedimento automático computacional. Elas se baseiam principalmente na antiga idéia proposta por Alan Turing em seu célebre artigo "Computing Machinery and Intelligence". Dessa forma, as HIP são elementos que reforçam a segurança de sistemas computacionais. A aplicabilidade das HIP para segurança da informação está na prevenção de determinados "ataques" que dependam da automatização. Exemplos destes tipos de ataques são: o envio de mensagens eletrônicas em massa (SPAM), votação automática em enquetes online, quebra de senhas via força bruta. O CAPTCHA, Completely Automated Turing Test for Telling Computers and Humans Apart, ou teste de Turing completamente automático e público para distinguir humanos de computadores, é a principal modalidade de HIP utilizada na Word Wide Web (WWW) atualmente. Um CAPTCHA possui alguns pontos chave que precisam ser observados: o teste tem que ser o mais amigável e simples possível para um ser humano médio; o teste tem que ser extremamente difícil, ou até impossível, para um computador; o teste tem que ser gerado automaticamente;	o teste deve possuir regras públicas; o teste não pode se basear em uma base de dados secreta como elemento integrante de sua estrutura, a lógica do teste não pode depender disso para atingir seus objetivos.
Dessa forma, um CAPTCHA utiliza problemas de Inteligência Artificial (IA) que se encontrem não resolvidos pela comunidade científica para estruturar sua lógica. Isso confere aos CAPTCHA's uma interessante propriedade: quando um CAPTCHA é quebrado, a IA avança em eficiência. Atualmente, para a construção de CAPTCHA's, utiliza-se problemas de IA relacionados à cognição humana visual e à cognição humana.. Situação: Concluído; Natureza: Pesquisa. Essa função é realizada por algoritmos e protocolos que são processados nos nós que a partir das informações coletadas pelo dispositivo que tomas as decisões de encaminhamento para manter o funcionamento da rede. Esse problema tem-se agravado nos últimos anos pelo fato do grande aumento das velocidade das interfaces óticas e das limitações de processamento dos processadores, tornando necessário o desenvolvimento de novas técnicas e mecanismos. Essa linha de pesquisa visa estudar, avaliar e propor novos protocolos e algoritmos de roteamento e encaminhamento de pacotes qua garantam confiabilidade e que sejam adequados às velocidades atuais..
