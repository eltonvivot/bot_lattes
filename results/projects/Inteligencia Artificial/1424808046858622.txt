2009 - 2011
Análise de Impacto e Localização de Feature no Contexto de Evolução de Software
Descrição: A presente proposta de pesquisa tem duplo objetivo. O primeiro objetivo é investigar e desenvolver técnicas e ferramentas de análise de impacto que apresentem resultados mais consistentes, reduzindo o número de falso-positivos e falso-negativos. O segundo objetivo é o de investigar causas que dificultam a compreensão de software durante a sua evolução e analisar e desenvolver técnicas de localização de feature que auxiliem na análise de impacto de mudanças. Com base nos objetivos acima declarados, definimos as seguintes metas/resultados esperados para o projeto: Meta 1. Investigação e definição de uma técnica de análise de impacto que reduza o número de falso-positivos e/ou falso-negativos; Meta 2. Investigação e definição de técnicas de localização de features que auxiliem no processo da análise de impacto de mudanças em software; Meta 3. Construção de suporte ferramental para validação das técnicas propostas. Meta 4. Aplicação prática da técnica desenvolvida; Meta 5. Estudo comparativo entre o trabalho proposto e trabalhos relacionados já existentes.. Situação: Concluído; Natureza: Pesquisa. 2008 - 2009
Smart Diagnostics
Descrição: The Smart Analysis project aims to specify, implement and test a software system to produce diagnostics of faults in power systems equipment and systems. The software system must be based on behavior models of such equipments and systems. Given a functional specification of an equipment, mathematical (or other types of) models of the equipment must be constucted and diagnostic techniques must be researched based on these models. Validation will be performed using complex electronic and electrical equipments in CHESF's laboratories. The project must base the models on a library of plug-in components (sub-models, functions, etc.), allowing the easy construction of complete models. The library must be sufficiently generic to model various equipment types, including digital models, analogical models, discrete models, probabilistic models, Petri nets, etc. The diagnostic precision must be sufficient to allow pinpointing faults down to the level of the Logical Replacement Units (LRUs). This project is financed by CHESF/Aneel.. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (4)  / Mestrado acadêmico: (2) . 2008 - 2009
Smart Alarms 2
Descrição: The Smart Alarms project aims to develop a software system to help power system operators diagnose system faults during large scale contingencies. The basic problem to be solved is event/alarm correlation in order to map hundreds or thousands of events/alarms into much fewer high-level diagnostics. The basic value proposition for the end product is that it is "zero maintenance": there is essentially no maintenance activity to be performed on the Smart Alarms system even in the face of major topology changes. The system is interfaced to CEPEL's SAGE SCADA/EMS system but can be interfaced to other such systems. The system also must provides the identification of the root -cause of events. This project is financed by CHESF/Aneel.. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (5)  / Mestrado acadêmico: (2) . 2003 - 2005
Mobile: Técnicas e Ferramentas para o Desenvolvimento e Vali
Descrição: Neste projeto nos propomos a investigar a adequação da utilização de métodos formais no desenvolvimento de software baseado em agentes móveis. Mais especificamente, nos propomos a utilizar uma classe de redes de Petri orientadas a objetos (RPOO) na modelagem, análise e verificação de sistemas de software, em especial aqueles com características de mobilidade. Acreditamos que a integração da teoria de concorrência (baseada em redes de Petri) e a modelagem orientada a objetos se constitui em uma poderosa ferramenta para os desafios que são encontrados no desenvolvimento prático de software com conceitos avançados como mobilidade. Nos propomos ainda a desenvolver ferramentas, métodos e técnicas que permitam a utilização de RPOO em cenários práticos.. Situação: Concluído; Natureza: Pesquisa.
