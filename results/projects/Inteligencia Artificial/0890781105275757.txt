2006 - 2009
SOLO: Structuring and Organizing Learning Objects
Descrição: Atualmente o desenvolvimento de materiais de aprendizagem segue a filosofia de orientação a objetos, originando os chamados "Objetos de Aprendizagem" ou "Learning Objects - LOs". Entretanto, não há uma definição amplamente aceita para o termo e nem uma metodologia de desenvolvimento amplamente adotada. Geralmente considera-se LOs como arquivos multimídia que foram desenvolvidos com enfoque no reuso e são utilizados para ensino, treinamento e/ou aprendizagem. Este projeto visa o desenvolvimento de um modelo conceitual para estes LOs de modo a promover seu reuso. Para isto procuramos incentivar a colaboração entre instituições e organizações de ensino através do compartilhamento de seus LOs, trabalhar com repositórios de LOs e de seus metadados, bem como adaptar documentos digitais disponíveis em bibliotecas digitais de modo que se tenha uma visão conceitual integrada destes elementos e portanto alavancar a educação à distância. A abordagem a ser adotada baseia-se em técnicas de integração de dados, principalmente aquelas relacionadas a camadas de esquemas e seus mapeamentos. A Web Semântica é vista como um agente capacitador de nossa proposta. Assim, trata-se de um projeto inovador e multidisciplinar, enfocando principalmente as áreas de Informática na Educação, Banco de Dados e Sistemas de Informação. Dentre os tópicos de pesquisa a serem trabalhados podemos citar: gerência de conhecimento, ontologias, serviços web, metadados, integração de dados, objetos de aprendizagem, composição de objetos e bibliotecas digitais.. 2000 - 2001
KIWI - Key technologies for the Integration of Web Information
Descrição: Este projeto tem como objetivo principal desenvolver uma infraestrutura, com tecnologias de base, para prover serviços de integração de informações na Web. As técnicas envolvem o gerenciamento de: metadados, workflow, replicação de dados, dados semi-estruturados e processamento de consultas. Este projeto tem como base para arquitetura de integração, o sistema de middleware Le Select, desenvolvido no INRIA. 
Interações anteriores bem sucedidas entre as equipes do Brasil e da França, apontam para a troca significativa de experiências entre as duas equipes no presente projeto. Os serviços serão desenvolvidos na forma de protótipos, passíveis de incorporação às arquiteturas de integração na web em geral. Os resultados serão validados em aplicações científicas como por exemplo o monitoramento de biofenômenos causadores de corrosão na produção de petróleo no CENPES/Petrobrás e controle de poluição aérea no projeto europeu Decair. Além disso, os serviços básicos gerados são também úteis à aplicações como educação a distância, trabalho cooperativo distribuído e comércio eletrônico de um modo geral. As tecnologias geradas no projeto podem ser transferidas para a indústria através de novos serviços para a web, desta forma contribuindo para o surgimento da sociedade de informação.
Dentre os principais aspectos tratados no presente projeto, enfatiza-se a abordagem descentralizada adotada para a integração de informações, de extrema relevância e adequação no contexto do volume cada vez maior de informações sendo disponibilizadas por órgãos e agências governamentais e mesmo instituições privadas. Nessa linha, a grande ênfase dada à gerência de metadados e à análise de padrões e ontologias, dando suporte à descrição, interpretação e recuperação desses recursos, procura dar um maior impulso ao tema, no contexto da Web. Por outro lado, a capacitação no tratamento de dados semi-estruturados e XML, acompanha as tendências evidenciadas pelos novos produtos sendo disponibilizados. 2000 - 2001
CoDIMS - Um Sistema Configurável para a Integração de Dados Heterogêneos e Distribuídos
Descrição: A proposta do Projeto CoDIMS - Configurable Data Integration Middleware System é fornecer um sistema constituído por frameworks componentes que, quando instanciados definem um sistema configurado para a integração de dados heterogêneos e distribuídos. Diversos sistemas são encontrados na literatura para integração de dados, mas geralmente são construídos para atender a uma aplicação específica ou procuram atender a várias situações diferentes, se tornando grandes e pesados. Nossa proposta visa a utilização de um mecanismo para configuração que permita construir sistemas com características específicas com um menor número de componentes possível e usando a técnica de reuso de software e de frameworks. 
Nos anos 90, a partir dos padrões de interoperabilidade do Object Management Group (OMG) e Object Database Management Group (ODMG), foi desenvolvido por este mesmo grupo de pesquisa o projeto HEROS (Heterogeneous Object System) para a integração de bancos de dados heterogêneos. Na última versão, a arquitetura do HEROS foi definida usando a técnica de frameworks, com o objetivo de proporcionar uma alternativa mais flexível e extensível para a criação e utilização de SGBDHs. Mas pela característica de sua arquitetura, todos os SGBDHs instanciados a partir do HEROS têm que apresentar todos os componentes, mesmo sendo desnecessários, acarretando os problemas dos sistemas heavy weight. Esta foi a maior motivação para a nossa atual proposta, de se pesquisar e implementar um sistema configurável, através de composição de frameworks. Composição de frameworks é um assunto atual e aberto na comunidade científica.

Projeto Tecnologias para Integração de Informações Heterogêneas com Aplicação em CAD e SIG
Descrição: A investigação proposta tem por finalidade a definição e criação de instrumentos que, apoiados na arquitetura CORBA, permitam a integração de informações dispersas em diferentes sistemas com o objetivo de apoiar o desenvolvimento de aplicações de CAD e SIG. 
Pretende-se também contribuir para a definição de um modelo de transações compatível com as necessidades dos sistemas de CAD e SIG.
O atual desenvolvimento tecnológico trouxe à luz a questão da existência de informações dispersas por diversos sistemas de características muito diversas, quer ao nível das plataformas de Hardware e de Software, quer à forma de representação e significado dos dados armazenados, motivando assim, a procura de instrumentos que permitam a integração destas informações nomeadamente para desenvolvimento de sistemas nas áreas de aplicação propostas: CAD e SIG. 
Por outro lado nas Intranets, navegadores WWW assumem papel semelhante ao de uma interface de utilizador genérica, permitindo o acesso a informações mantidas não somente em documentos no formato HTML, como também em arquivos convencionais, bancos de dados e outros tipos de fontes. No entanto, permanece ainda a dificuldade de oferecer ao usuário uma visão integrada dos diferentes tipos de dados, através de um sistema integrador de informações.
O uso de padrões para a orientação a objetos tem sido considerado como a solução para coordenar, com flexibilidade, e integrar os recursos de processamento de informações, interligados por redes de comunicação. Na arquitetura dos sistemas de bancos de dados heterogêneos, estes padrões podem ser utilizados para a integração e interação dos sistemas heterogêneos componentes do sistema visto globalmente, aumentando sua flexibilidade para adaptar-se a novos ambientes.
Assim, o recurso à especificação CORBA poderá viabilizar a construção de um ambiente capaz de contribuir para a resolução destas questões.

1998 - 1999
ECOHOOD - Environment of Configurable Heterogeneous Object-Oriented Database
Descrição: --	Novos tipos de aplicações, tais como, multimídia, workflow, data warehousing e aplicações baseadas na web requerem funcionalidades diferentes para um Sistema Gerenciador de Banco de Dados (SGBD). A extensão de SGBDs de propósitos gerais que atenda a estes requisitos introduz várias dificuldades de administração, desenvolvimento e performance. Por outro lado, a produção de SGBDs específicos de acordo com as necessidades das aplicações implica em um trabalho de desenvolvimento. Este projeto apresenta um enfoque alternativo, que é a configuração de SGBDs. Um SGBD configurado consiste em uma seleção de serviços inter-relacionados requeridos pelo gerenciador de dados num domínio de aplicação específico. Os serviços usados numa configuração envolvem não apenas as funcionalidades tradicionais do SGBD, como também, as não tradicionais, tais como sincronismo de gravação de áudio e vídeo necessários numa aplicação multimídia. No projeto ECOHOOD foi proposto um novo enfoque para a construção de SGBDs baseado nos conceitos de frameworks da engenharia de software, que define uma arquitetura orientada a objeto suportando reuso. Palavras-chave: construção de SGBD, extensibilidade de um SGBD, frameworks.. Situação: Desativado; Natureza: Pesquisa. Ecobase - Tecnologias de Banco de Dados e Web para Sistemas de Informações Ambientais
Descrição: --	Um número muito grande de fontes de dados sobre meio ambiente e recursos naturais está disponível no mundo todo. Infelizmente, usuários desses dados geralmente enfrentam vários problemas quando querem buscar e usar informações ambientais. O projeto ECOBASE vem abordando esses problemas para facilitar o acesso a informações ambientais. Uma arquitetura comum baseada em componentes de Web foi proposta para sistemas de informações ambientais (SIAs). Essa arquitetura consiste de um "data warehouse" de dados ambientais que pode ser alimentado a partir de várias fontes de dados heterogêneas distribuídas na Internet.  Os temas de pesquisa envolvidos incluem as tecnologias de "data warehouse" e mediadores, gerência de metadados, gerência de dados espaciais, regras ativas, replicação, e qualidade da informação. Os participantes do projeto estão todos trabalhando em vários aspectos de SIAs e em aplicações ambientais específicas, compartilhando seus resultados de pesquisa, componentes e experimentações no Brasil e na Europa. HEROS (Heterogeneous Object System)
Descrição: -- 
HEROS é um SGBD distribuído baseado em um modelo de objetos que integra BDs heterogêneos. Nesse sistema, integra-se BDs implementados com SGBDs diferentes (diferentes modelos, diferentes SGBDs do mesmo modelo, etc.). O sistema adota a arquitetura cliente/servidor onde os clientes interagem com um servidor HEROS global, que tem um esquema orientado a objetos resultante da integração dos esquemas dos BDs componentes. O servidor HEROS global, por sua vez, é cliente dos servidores HEROS locais. Os servidores locais são responsáveis pela comunicação com os SGBDs componentes. A comunicação entre clientes e servidores utiliza o protocolo TCP/IP e chamadas a procedimentos do tipo RPC.. Situação: Desativado; Natureza: Pesquisa.
