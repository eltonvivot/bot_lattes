Análise de avaliações de acessibilidade digital publicadas na Google Play Store associadas a deficiências visuais
Descrição: Avaliações de usuários publicadas em lojas de aplicativos ajudam organizações a corrigir defeitos e planejar novas versões de aplicações. Em particular, avaliações relacionadas à acessibilidade podem ser úteis para entender as barreiras que os usuários enfrentam em ambientes reais. Estudos que analisam avaliações que tratam de acessibilidade conclu1ram que poucos usuários mencionam acessibilidade em suas avaliações e que poucos tipos de barreiras são mencionados. Entretanto, esses estudos possuem algumas limitações: amostra pouco representativa, extra1das de aplicações pouco utilizadas e com cerca de 200 mil avaliações no total, e análises fundamentalmente quantitativas, que não exploram os principais conceitos e seus relacionamentos abordados pelos usuários. O objetivo deste projeto de pesquisa é caracterizar as questões de acessibilidade abordadas em avaliações de acessibilidade e que afetam usuários com deficiência visual. Esta caracterização será feita por meio da aplicação de técnicas quantitativas e qualitativas aplicadas sobre uma amostra representativa com aproximadamente 180 milhões de avaliações. Os resultados desta pesquisa irão contribuir com a área ao i) fornecer evidências sobre as barreiras de acessibilidade que afetam a percepção, a compreensão e a operação das aplicações em ambientes reais; ii) identificar as principais barreiras de acessibilidade enfrentadas para cada tipo de deficiência ou elemento da interface; iii) identificar o impacto das barreiras para os usuários com base nas expressões utilizadas por eles, análise de sentimentos e notas atribu1das à aplicação; iv) sistematizar o conhecimento extra1do das avaliações para fornecer evidências sobre a importância de seguir guias de acessibilidade.. 2016 - 2017
Automatic Test Suite Generation for Mobile Applications Using Search-Based Testing
Descrição: Mobile applications represent an already tremendous, but still quickly growing market. In a world where users have ample choice of apps in overcrowded app stores, every bug can be the demise of an app. The scale of economic damage caused by software problems in mobile apps is huge, and is only overshadowed by the threat posed by increased usage of apps in safety relevant scenarios. The traditional answer of software engineers to these software bugs is software testing. However, mobile apps pose challenges over the already complex and error prone task of software testing. Automation is desirable to overcome these challenges, yet existing techniques to automate test generation struggle with complex user interactions in mobile apps (e.g. input data, tap, double tap, long press, gestures), interactions with the environment (e.g. sensors, camera, GPS), external events (e.g. other applications of OS), and multiple configurations (e.g. different devices and OS properties). In order to handle the complexity of the problem of automating tests for mobile apps, we propose to develop a search-based approach, where tests consisting of complex user and environment interactions will be generated automatically, guided by search heuristics based on functional as well as structural criteria. The availability of such a search-based testing framework will support app developers and app store maintainers in increasing the quality of their products, and it will serve as foundation for future research on mobile apps. The ambitious goals of this research project are made feasible by a combination of the experience of the principal investigator and the host institution, which has a track record in successful development of search-based approaches.. 2014 - 2016
Geração automática de dados de teste para cobrir caminhos dependentes de exceção usando execução simbólica
Descrição: O teste de software é uma atividade chave para a garantia de qualidade de software. Técnicas e estratégias de teste auxiliam os testadores na criação e na avaliação de casos de teste. A execução simbólica tem sido utilizada como uma técnica efetiva para a geração automática de dados de teste para cobrir os requisitos de teste definidos pelo critério de fluxo de controle da técnica de teste estrutural. Na execução simbólica, cada caminho do programa sob teste é representado por uma sequência de restrições que deve ser satisfeita palas variáveis de entrada para que o caminho seja executado.
Valores concretos são definidos para cada variável por meio de solucionadores de restrições (constraint solvers), tal que todas as restrições sejam satisfeitas. Os valores das variáveis são então usados como dados de teste para executar cada caminho.
A maioria das abordagens explora apenas técnicas para gerar dados de teste para caminhos livres de exceção, que são caminhos nos quais os mecanismos de tratamento de exceção não são levados em consideração. A geração de dados de teste para caminhos dependentes de exceção, que são os caminhos que são executados apenas quando uma dada exceção é lançada, não tem sido tratada apropriadamente pelas abordagens atuais. Essa limitação existe porque as restrições para que uma exceção seja lançada não estão sempre explícitas no código, a não ser que sejam declaradas pelos desenvolvedores. Se não houver restrições definidas pelo usuário, somente uma análise do código sob a ótica de cada tipo de exceção pode revelar quais dados de teste podem provocar a exceção. 
Portanto, este projeto de pesquisa tem o objetivo de investigar a geração de dados de teste para caminhos dependentes de exceção e propor técnicas para definir restrições capazes de garantir que as exceções necessárias sejam lançadas e evitar que as exceções indesejadas sejam ativadas..
