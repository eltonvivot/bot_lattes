SERIN Discovery
Descrição: SERIN é uma especificação de interfaces semânticas para Serviços Web RESTFul que prevê uma padronização para a URL dos Serviços Web. &#10;&#10;Esta padronização permite ao Servidor de Descoberta SERIN utilizar um web crawler para descobrir que hosts implementam uma específica interface semântica. &#10;&#10;A arquitetura apresentada incorpora mais três vantagens em relação a dos servidores&#10;UDDI: &#10;1) É baseada em interfaces abstratas aderentes a Web Semântica; &#10;2) Possui maior escalabilidade pois não exige que os desenvolvedores registrem seus&#10;serviços e, finalmente, &#10;3) possui uma interface semântica RESTFul de consultas que permite a agentes de software automatizar consultas ao servidor de descoberta seguidas da subsequente execução dos serviços descobertos.. SERIN - Semantic REST Interfaces
Descrição: Interfaces são abstrações largamente utilizadas em todos os níveis da computação e exercem, tradicionalmente, papel fundamental na interconexão de sistemas de computação.&#10;&#10;No contexto da Web Semântica, adaptadando o conceito de Ontologias ativas para sua aplicação a Serviços Web RESTFul, SERIN propõe a criação de interfaces semânticas abstratas indicando quais serviços web RESTFul deveriam ser construidos por quem implementar a interface.&#10;&#10;A especificação da Interface Semântica é uma ontologia escrita em OWL acrescida de anotações indicando, apara cada calsse e/ou proprieade quais, entre cinco tipos de serviços web (GET,PUT,POST, LIST,DELETE), estarão disponíveis.&#10;&#10;SERIN utiliza uma convenção de endereçamento para os serviços web que permite localizá-los conhecendo o host e a interface semântica utilizada.&#10;&#10;O Projeto SERIN é uma continuação do projeto de Ontologias Ativas que se aplicava a Web services semânticos Soap-based. Situação: Em andamento; Natureza: Pesquisa. Alunos envolvidos: Graduação: (1)  / Mestrado acadêmico: (4)  / Doutorado: (1) . 2004 - 2008
GODEL
Descrição: O tema do projeto é a exploração de métodos matemáticos para a solução de problemas complexos. Por complexos, de uma forma precisa, entendem-se problemas cuja solução em tempo polinomial é desconhecida, possivelmente inexistente. Duas são as abordagens básicas para solucionar problemas complexos e executar tarefas sofisticadas em computação: a abordagem teórico-simbólica e a abordagem algorítmica. Pela primeira, entendemos a elaboração de teorias utilizando a linguagem e as técnicas de inferência da lógica matemática na representação e solução de problemas. Na segunda abordagem, algoritmos computacionais são especialmente confeccionados para implementarem as tarefas requeridas pela especificação dos problemas. Estas duas abordagens apresentam claros pontos de encontro, sendo, na realidade de natureza complementar: as soluções formuladas em termos lógicos carecem de implementação algorítmica, assim como algoritmos podem beneficiar-se da análise lógica no estudo e determinação de suas propriedades, em particular sua correção. Por exemplo, a especificação dos problemas pode ser feita em linguagens formais e os algoritmos são implementados em uma linguagem de programação declarativa, tendo sua correção com respeito a essa especificação analisada por instrumentos lógicos. A competência do núcleo abrange três linhas que compõem a área de Teoria da Computação e Algoritmos, a saber: (i) Lógica e Inteligência Artificial, (ii) Otimização Combinatória, e (iii) Aplicações de Métodos Matemático-Formais na Solução de Problemas Complexos. A experiência dos pesquisadores se reflete nas publicações científicas relevantes em periódicos indexados nos temas do Projeto GÖDEL, pelos projetos de pesquisa financiados e com inserção nacional e internacional, pela quantidade e qualidade de orientações na pós-graduação stricto sensu. As experiências individuais nos temas propostos podem ser comprovadas nos currículos LATTES dos pesquisadores... Situação: Em andamento; Natureza: Pesquisa.
