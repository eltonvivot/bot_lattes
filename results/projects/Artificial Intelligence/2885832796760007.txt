2019 - Atual
Big Data and Business Intelligence - Qualidade de Dados em diferentes cenários -  problemas e soluções
Descrição: Este projeto visa avaliar problemas de qualidade de dados nos contextos da saúde e de biodiversidade e propor soluções apropriadas para mitigar os problema de qualidade de dados identificados.  No contexto da saúde, códigos de diagnósticos mal documentados causam erros nas classificações de pacientes e hospitais e no embasamento de modelos de custos do paciente.  Essas classificações e modelos de custos são baseados na codificação de diagnósticos secundários, que é feita geralmente de forma manual e com base em registros clínicos e laudos de procedimentos. O processo manual é lento, propenso a erros e exige grande esforço. Além disso, há evidência de problemas de qualidade nos dados de registros clínicos como ausência e ambiguidade. O tratamento da qualidade dos dados clínicos e de laudos de procedimentos são essenciais para a automação bem-sucedida da codificação automática dos diagnósticos secundários.  No contexto da biodiversidade, modelos de distribuição de espécies se tornaram uma ferramenta importante na gestão de conservação. No entanto, problemas de qualidade nos dados utilizados na modelagem de distribuição de espécies resultam em modelos imprecisos e que não refletem o real padrão da distribuição. Como consequência, estratégias de conservação baseadas em modelos de distribuição gerados por dados enviesados, podem resultar no desperdício de recursos financeiros ou perda importante de biodiversidade.. 2017 - 2020
Big Data e Businesss Intelligence - uso de dados de redes sociais como solução para cenário de cold-start em sistemas de recomendação
Descrição: Nos últimos anos aumentaram vertiginosamente a geração de dados e as oportunidades associadas com sua análise. Muitos desses dados estão vindo de redes sociais, blogs, fotos e vídeos, além de servidores e/ou sistemas que gravam logs de navegação e qualquer ação relevante de usuários que estão visitando a web. O termo Big Data  reflete as características destes dados e os requisitos para sua gestão, a qual é mais complexa, dado o volume, a variedade e a velocidade necessária para o processamento desses dados. Tirar proveito dessa enorme quantidade de dados é uma tarefa extremante complexa para o ser humano sem o auxílio computacional.  Sistemas de recomendação tem o propósito de auxiliar o usuário a encontrar artefatos (produtos, documentos, informações, serviços) que podem interessá-lo.  Como a recomendação se baseia em dados sobre o usuário, em cenários de cold-start, onde não existem informações disponíveis sobre o mesmo, a recomendação se torna um problema de maior complexidade.  O problema de cold-start tem recebido bastante atenção e várias propostas têm sido apresentadas na literatura, contudo ainda não existe uma solução que possa ser aplicada a qualquer domínio ou situação. Neste cenário, dados extraídos de redes sociais são uma fonte de informação valiosa que possibilita recomendações assertivas para o usuário.  Este projeto apresenta uma solução para o contexto de cold-start utilizando somente dados extraídos de redes sociais.. Situação: Concluído; Natureza: Pesquisa. Qualidade de dados em ambientes de negócio
Descrição: quando necessitamos, por exemplo, unir dados espalhados em diversos meios de armazenamento (planilhas, SGBDs, arquivos, etc.) na tentativa de gerar um banco de dados corporativo dentro de uma empresa, ou coletar dados vindos de uma ou mais fontes para a geração de um armazém de dados (Data Warehouse), ou de um sistema de CRM (Customer Relationship Management), para a extração de informações  estratégicas  para o futuro de um empreendimento, ou ainda quando precisamos migrar dados dentro de um processo de manutenção de dados (ISO/IEC 1995) na substituição de um sistema existente, em geral, nos deparamos com as seguintes indagações: qual a acurácia do dado que está registrado? Quão correto ele é? Como ele foi gerado? Qual o seu real significado? Para responder a estes questionamentos métodos, processos e ferramentas têm sido desenvolvidos como produto das pesquisas. Contudo, faltavam muitas vezes dados para seu emprego em ambientes reais. Este projeto visou a avaliação do emprego real de algumas dessas metodologias em ambientes de negócio com o intuito de validar benefícios e propor melhorias aos processos para facilitar sua utilização.. 2003 - 2013
Orientação a objetos e banco de dados
Descrição: O objetivo foi pesquisar o tratamento de objetos em banco de dados, incluindo o mapeamento de objetos da aplicação para objetos de banco de dados, além dos recursos existentes em SGBDs para manipulação de objetos nos gerenciadores existentes de forma a eliminar o descasamento de impedância existente entre a camada de aplicação e a base de dados.
A pesquisa explorou  a utilização dos novos tipos de dados e recursos disponíveis em SGBDOR, incluindo formas de mapeamentos apropriados entre modelos de dados conceituais e o modelo lógico para SGBDORs e vice-versa. Existe uma definição muito clara e padronizada quando se faz o mapeamento para SGBDRs dos conceitos, objetos e restrições de um Universo de Discurso (UoD) representados por um modelo de conceitual, seja um Modelo Entidade-Relacionamento (MER), ou um modelo de classe descrito pela Unified Modelling Language (UML). Porém, o mesmo não é observado para o contexto de SGBDORs.  Assim, a pesquisa considerou os novos tipos de dados e as características especificadas pela SQL:2003 e que estão presentes num SGBDOR. Outro resultado foi um Framework para mapeamento de objetos da aplicação em objetos de banco de dados.. Situação: Concluído; Natureza: Pesquisa.
