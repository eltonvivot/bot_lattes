Modelo matemático para obtenção de propriedades mecânicas de ligas de alumínio
Descrição: Para realização de qualquer projeto mecânico torna-se necessário conhecer as características do material que será utilizado e projetar o equipamento, de tal maneira que qualquer deformação resultante não seja excessiva e não provoque fratura. As propriedades mecânicas dos materiais são verificadas pela execução de experimentos ou ensaios de laboratório cuidadosamente programados, que reproduzem o mais fielmente possível as condições de serviço. Entretanto, estes ensaios mecânicos são destrutivos. Esta característica destrutiva do ensaio pode ser um problema quando as peças que precisam ser testadas não podem ser danificadas. Este é o caso, por exemplo, de peças que já passaram por vários processos de fabricação como, fundição, acabamento, usinagem e montagem, ou que já estejam em funcionamento por um período de tempo, e por algum motivo, precisam ter suas propriedades mecânicas determinadas. Um outro complicador é a determinação de propriedades mecânicas em pontos específicos da peça. Esta necessidade se justifica principalmente em peças fundidas onde as condições de resfriamento, as microssegregações e a ocorrência de defeitos se distribuem de maneira heterogêneas ao longo da peça. Uma vez que o processo de determinação da composição química e da estrutura metalográfica é realizado através de ensaios não destrutivos, estes poderiam substituir os ensaios mecânicos destrutivos, se fosse estabelecida uma relação entre estes. É de conhecimento que existe uma relação entre a composição química, a estrutura metalográfica das ligas e as suas propriedades mecânicas. Entretanto, esta relação não é direta e dependem de muitos fatores complexos de serem combinados, inclusive defeitos inerentes ao processo de fabricação por fundição. Este trabalho apresenta um modelo matemático para a determinação das propriedades mecânicas de ligas de alumínio fundidas, hipoeutéticas e eutéticas, a partir da realização de ensaios não destrutivos de microscopia e análise química.. Aplicação da rede self-organizing map supervisionada para classificação de elementos finitos
Descrição: O Método dos Elementos Finitos pode incorporar diferentes teorias e modelos para o tratamento de um determinado problema de engenharia. Os diversos programas de elementos finitos existentes, embora possam ser empregados na resolução de problemas semelhantes, apresentam elementos próprios, e em formatos proprietários, muitas vezes originários de formulações distintas, para resolução de um mesmo problema. A linguagem XML se apresenta como facilitadora no processo de importação e exportação de dados entre os diferentes programas de elementos finitos utilizando o esquema FEML proposto por Pinheiro e Moita em 2003. Este trabalho apresenta resultados de uma ferramenta, baseada na tecnologia das Redes Neurais Artificiais, especificamente da rede Self-organizing Map (SOM), com o emprego do algoritmo Learning Vector Quantization, implementada para fazer a classificação das famílias dos elementos finitos para uso na FEML. Esta ferramenta realiza a classificação, em termos de elementos equivalentes ou semelhantes (menos especializados), dos elementos provenientes de programas distintos em um procedimento de exportação e importação de dados em elementos finitos. Através da auto-organização do mapa das características comuns recebidas, obtida a partir de um conjunto de classes informadas pelo supervisor da rede, a rede SOM é capaz de encontrar a equivalência entre os padrões, obtendo como resposta a classe que representa o agrupamento dos padrões por ela identificados. Proposta de um exquema XML para o intercâmbio de dados em análise por elementos finitos
Descrição: No universo dos softwares de análise de estruturas que se utilizam do Método dos Elementos Finitos não há um formato padronizado para o intercâmbio de dados. Isso dificulta o trabalho com mais de um aplicativo para a solução ou modelagem de um problema qualquer. A existência de um padrão poderia agilizar o processo de análise na medida em que se pudesse utilizar um padrão de saída para alimentar a entrada de outro sistema, além da diminuição de erros no processo de transcrição de dados. A proposta da XML - Extensible Markup Language - permite não apenas a estruturação de dados, mas também sua validação através da criação de vocabulários, também denominados esquemas (schema), que podem ser utilizados para garantir a correção e integridade dos dados e de sua estrutura. Este estudo propõe a criação de um vocabulário XML para a descrição de dados que discretizam estruturas a serem modeladas a partir do Método de Elementos Finitos. Desta forma, com pequenas implementações nas interfaces de entrada e saída de cada sistema, garantir-se-á um método muito mais rápido e confiável de intercâmbio de dados entre esta classe de software.. Aplicações Web em XML: estágio atual e tendências futuras
Descrição: Neste trabalho, foi estudado o ambiente de desenvolvimento Web e principalmente, a XML (eXtensible Markup Language ou Linguagem de Marcação Extensível), linguagem que desponta como a melhor solução para a estruturação dos dados que devem ser trocados e transferidos entre sistemas de plataformas distintas na Internet. Com o crescimento quantitativo e qualitativo da Internet, as linguagens e tecnologias utilizadas atualmente tornam-se pouco adequadas para as várias necessidades advindas deste aumento de demanda. O problema é relativamente bem delineado: o crescente esgotamento da HTML como única linguagem para dispor dados resultantes de aplicações Web, além da necessidade de um novo recurso para troca eletrônica de dados de forma direta, entre sistemas e plataformas diferentes. Ao aprofundar o estudo na linguagem XML, seu potencial e as ferramentas de suporte a ela, este trabalho pretende contribuir para uma melhor compreensão desta tecnologia, servindo de base para estudos posteriores que possam determinar e conhecer mais profundamente a XML..
