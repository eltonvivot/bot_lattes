Descrição: No contexto de Sistemas Colaborativos (SC) a abordagem de ecossistema de software se

necessidade de oferecermos o suporte à interoperabilidade se torna um requisito essencial. Isso

tecnologias, em contextos organizacionais específicos, bem como culturas e termos de seus

colaboração na realização de experimentos científicos geograficamente distribuídos, de forma
eficiente e eficaz, a interoperabilidade é um problema para os desenvolvedores e usuários de

ambiente, ou ao tratamento não automatizado da troca de dados quando ferramentas distintas

podem ser implicitamente tratados na própria aplicação, gerando sistemas fortemente acoplados.
Portanto, não basta tratarmos apenas os aspectos sintáticos para o suporte à interoperabilidade,
mas sobretudo elementos mais complexos para apoiar outros níveis de interoperabilidade mais
abstratos, tais como, semânticos e pragmáticos. Este projeto tem como objetivo avançar no
estado da arte em relação às soluções de interoperabilidade entre SC. Além disso, buscamos
propor e avaliar soluções para os diferentes níveis de interoperabilidade no contexto de uma
plataforma, que utiliza a abordagem de ecossistemas de software, para apoiar o desenvolvimento

MODDEN - Uma Plataforma de Suporte à Engenharia de Domínio Dirigida a Modelos
Descrição: Este projeto tem como objetivo especificar um processo que integra a Engenharia de Domínio (ED) e o Desenvolvimento Dirigido por Modelos (DDM), para construção de artefatos (modelos, componentes, serviços, etc.) que possam ser reutilizáveis em aplicações de um mesmo domínio. Além disso, objetiva também o desenvolvimento de uma ferramenta que apóie o uso deste processo por equipes de desenvolvimento de software. Técnicas de reuso tornaram-se uma constante diante de cenários que demandam produtividade, baixo custo e qualidade. Construir softwares com qualidade e produtividade tem sido decisivo para a sobrevivência das empresas desenvolvedoras de sistemas. Neste contexto, abordagens vêm sendo propostas ao longo dos anos, tais como: orientação a objetos, desenvolvimento baseado em componentes, arquitetura orientada a serviços, entre outras. A Engenharia de Domínio busca apoiar o projeto e a implementação de uma infra-estrutura de reutilização de componentes e interfaces, o funcionamento e a comunicação entre esses componentes. O Desenvolvimento Dirigido por Modelos (DDM) utiliza modelos como uma base para a construção de um software, de forma que eles sejam os artefatos principais deste processo. Nesta abordagem, modelos não são usados apenas como documentação de sistemas, mas principalmente como uma ferramenta para a implementação. Desta forma, observa-se que na ED o reuso é considerado desde a concepção dos artefatos, e que uma vez implementados, formam uma biblioteca disponível para ser reutilizada por diversas aplicações do domínio. A DDM permite, através de transformações automatizadas, iniciar uma especificação em alto nível de abstração e gerar o
código específico para uma plataforma de execução. Com a integração destas duas abordagens pretende-se produzir artefatos reutilizáveis com maior produtividade, flexibilidade e portabilidade. WGWSOA: Uma Arquitetura Orientada a Serviços de Middleware para Apoiar o Desenvolvimento de Sistemas Colaborativos
Descrição: Aplicações distribuídas de groupware requerem um suporte à interoperabilidade entre plataformas com o intuito de potencializar o projeto, a implementação e a evolução de suas funcionalidades. Groupware distribuído, necessitam de requisitos difíceis de serem atendidos por uma única aplicação. Essa dificuldade está relacionada tanto à quantidade de requisitos funcionais como à diversidade de requisitos não-funcionais que tais aplicações demandam. A ausência de interoperabilidade é um problema tanto para os usuários como para os projetistas de groupware. A WGWSOA (Web-based Groupware Service-Oriented Architecture) é uma arquitetura orientada a serviços de middleware cujo objetivo é apoiar a reutilização e a interoperabilidade entre sistemas colaborativos.
