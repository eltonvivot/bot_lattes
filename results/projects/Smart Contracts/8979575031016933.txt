2012 - 2014
Integrando Padrões de Processos e Organizacionais para gerar Processos de Software
Descrição: Sabe-se que a qualidade é um requisito imprescindível para garantir a sobrevivência de um software no mercado. As empresas mais competitivas são aquelas que trabalham sob a ótica da melhoria contínua dos processos. Aumentando a qualidade do processo de desenvolvimento, conseqüentemente, se aumenta a qualidade do produto final. Dentro deste escopo, destaca-se a importância da padronização dos processos. Uma abordagem padronizada oferece um alicerce fundamental ao processo de desenvolvimento de software, garantindo que todas as etapas atinjam resultados previsíveis e de qualidade assegurada. Além disso, os padrões otimizam todas as etapas do processo de desenvolvimento de software, aumentando a produtividade e minimizando o re-trabalho. Eles também podem ser usados para representar ações preventivas e corretivas a riscos, por exemplo. Os produtos de trabalho padronizados são mais fáceis de serem compreendidos e avaliados. Neste contexto, o presente projeto de pesquisa visa gerar processos de software a partir de padrões usando Business Process Modeling Notation (BPMN) e posteriormente o objetivo será simular esses processos com a finalidade de avaliar custo e tempo. A principal contribuição do trabalho é que através da representação dos padrões de processo usando BPMN torna-se possível demonstrar uma maneira mais compreensível e precisa as características de processos de software. A simulação servirá para comparar processos e explorar os efeitos das variações entre os mesmos. O produto esperado dessa pesquisa é a representação e integração de padrões usando BPMN para elaboração de processos de software. Serão publicados artigos para disseminar os resultados obtidos na pesquisa e promover o desenvolvimento das autoras do projeto.. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (2) . Integrantes: Lisandra Manzoni Fontoura - Coordenador / Camila Hudner Brondani - Integrante / Darciele Aparecida Zilio de Souza - Integrante.

2007 - 2008
Elaboração de Ambientes para Execução de Processos de Software Adaptáveis
Descrição: Este projeto de pesquisa visa experimentar o uso de sistemas de gerência de workflow para elaboração de ambientes de execução de processos adaptáveis. Como resultado, será elaborado um protótipo de uma ferramenta para possibilitar a instanciação de ambientes configuráveis para uso em projetos específicos. Os processos são adaptados conforme descrito pela sistemática para gerenciar riscos em projetos de software (Project Risk Management Approach ? PRiMA). PRiMA tem como objetivo permitir a elaboração de um processo específico para um dado projeto, visando minimizar a exposição do projeto aos riscos, identificados de acordo com o contexto do projeto.Como resultado da adaptação de processo, é gerado um processo de software para ser utilizado em um projeto específico da organização. Os aspectos dinâmicos desses processos são descritos por meio de diagramas de atividades, e a descrição de cada atividade é armazenada em uma base de conhecimento, usando os conceitos papéis, artefatos de entrada e saída, ferramentas, e guias. Os diagramas de atividades podem ser convertidos em uma descrição em XPDL (XML Process Definition Language). XPDL possibilita descrever processos que, posteriormente, serão instanciados e executados em sistemas de gerência de workflow open source, que seguem o padrão de referência, proposto pela Workflow Management Coalision  (WfMC ).. Ambiente Experimental para Gerência de Projetos
Descrição: Esta pesquisa tem como objetivos principais estudar métricas usadas para gerenciar projetos de software orientados a objetos e estender o Software para Coleta e Análise de Métricas (SCAM) para gerenciar projetos orientados a objetos que utilizem a Linguagem de Modelagem Unificada (UML), como notação.
SCAM foi desenvolvido pelo orientador proponente deste projeto. SCAM é uma ferramenta que suporta a análise e a coleta
de métricas em projetos de projeto. SCAM considerada apenas as métricas de cronograma (tempo e esforço), extraídas de
um XML gerado pela ferramenta GANTT Project.
As métricas coletadas por SCAM são bastante importantes para o acompanhamento dos projetos, mas identifica-se que
métricas relacionadas ao produto, tais como: tamanho, qualidade, requisitos, entre outros; são também bastante
importantes.
Esse projeto de pesquisa visa também pesquisar outras formas de coleta de métricas, possivelmente o uso de sistemas
multiagentes ou compartilhamento de dados armazenados em um banco de dados, pois algumas métricas não são
possíveis de serem coletadas por meio de arquivos XML.
SCAM está sendo desenvolvido em Java e JSP, sendo portanto portável a diferentes ambientes experimentais.. Situação: Em andamento; Natureza: Pesquisa. 2004 - 2006
Ferramenta para Automatizar a Coleta e Análise de Métricas em Projetos de Software
Descrição: Essa pesquisa tem como objetivos principais estudar os tipos de métricas usadas para gerenciar projetos de software e desenvolver uma ferramenta para automatizar a coleta de métricas em ambientes de trabalho distribuídos. Os dados coletados serão analisados de forma a fornecer subsídios para o planejamento e o acompanhamento quantitativo dos projetos de software. 
A coleta e análise de métricas durante a execução do processo de software possibilitam o monitoramento dos projetos de software, a melhoria contínua do processo de desenvolvimento e a avaliação da qualidade do produto. Com base nos dados coletados, os gerentes de projeto podem fundamentar suas tomadas de decisão, o desempenho de profissionais pode ser comparado com seu histórico, avaliando impactos de treinamento,  ou uso de ferramentas e processos, projetos podem ser acompanhados e estimativas traçadas, o processo de estimativas pode ser melhorado, etc. 
Coletar métricas aderentes a um processo disciplinado é ainda uma tarefa difícil em qualquer projeto de software, principalmente se feita de maneira manual. Para gerenciar quantitativamente projetos de software é necessário fornecer suporte automatizado para gerenciamento de projetos de software e medição. Ferramentas automatizadas auxiliam o controle e o acompanhamento de projetos de software.
A ferramenta para automatizar a coleta de métricas deve ser portável aos diferentes ambientes operacionais existentes, portanto, será desenvolvida utilizando uma plataforma de desenvolvimento livre..
